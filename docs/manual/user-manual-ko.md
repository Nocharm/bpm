# Business Process Map — 사용자 매뉴얼

> Business Process Map(BPM)은 업무 프로세스를 플로우차트로 그리고, As-Is / To-Be **버전**으로 관리하며, **승인 워크플로**로 확정하는 웹 서비스입니다.

이 매뉴얼은 일반 사용자가 알아야 할 모든 내용을 다룹니다 — 첫 맵 작성부터 승인·게시까지. 시스템 관리자는 별도의 **관리자 매뉴얼**도 함께 읽어 주세요.

---

## 1. 시작하기

### 로그인

1. 앱에 접속해 **Sign in with Keycloak**을 클릭하면 사내 SSO 계정으로 로그인됩니다.
2. 개발 환경에서는 **Sign in with a test account**로 테스트 사용자를 골라 로그인할 수 있습니다.

### 화면 한눈에 보기

| 화면 | 위치 | 역할 |
| --- | --- | --- |
| **맵 목록(Maps)** | 상단 탭 | 전체 프로세스맵 — 검색·생성·열기 |
| **공지사항(Notices)** | 상단 탭 | 관리자가 게시한 공지 |
| **알림·승인(Inbox)** | 상단 탭 | 내가 처리할 승인 요청과 알림 |
| **사용 매뉴얼(Manual)** | 상단 바 | 매뉴얼 문서들(한/영) — 제목 드롭다운으로 선택, 목차와 본문 검색 지원 |
| **피드백(Feedback)** | 상단 바 | 버그·제안을 관리자에게 전송 |
| **설정(Settings)** | 상단 바 | 내 그룹, 삭제 예정(휴지통), 관리자 콘솔 |

> **팁:** 맵 목록과 이 매뉴얼에서 `/` 키를 누르면 바로 검색창으로 이동합니다.

### 맵에서의 역할

| 역할 | 할 수 있는 일 |
| --- | --- |
| **Owner** | 전부 — 편집, 협업자·결재자·공개 범위 관리, 삭제 |
| **Editor** | 맵과 초안 버전 편집 |
| **Viewer** | 읽기 전용 — 열람, 비교, 댓글 |
| **결재자(Approver)** | 승인 요청된 버전의 검토·승인/반려 |

공개 범위가 **공개(Public)**인 맵은 누구나 볼 수 있고, **비공개(Private)** 맵은 협업자와 결재자만 볼 수 있습니다.

## 2. 맵 목록 (홈)

### 맵 찾기

- **최근 열어본** 밴드에 마지막으로 작업한 맵이 표시됩니다 — **Show more** / **Collapse**로 펼치고 접습니다.
- **All maps**에는 내가 볼 수 있는 모든 맵이 나옵니다. `/` 키 또는 검색창(**Search maps**)으로 이름을 필터링하세요.
- 필터 칩으로 **상태**(초안 / 검토 대기 / 승인됨 / 반려됨 / 게시됨)와 내 **역할** 기준으로 좁힐 수 있습니다. **Clear filters**로 초기화합니다.
- 카드를 클릭하면 **상세 패널**이 열립니다: 소유자, 노드 수, 열람 가능 멤버(개인 / 팀 / 유저그룹), 버전별 이력(생성·승인요청·승인·반려·게시·회수).

### 맵 만들기

1. **New map**을 클릭합니다.
2. 이름과 설명을 입력하고 **공개 범위**(공개 / 비공개)를 선택합니다.
3. 필요하면 **초기 협업자**를 추가합니다 — 개인·팀·유저그룹을 Viewer 또는 Editor로.
4. **결재자를 1명 이상** 지정합니다 — 결재자 없이는 맵을 만들 수 없습니다.

### 복사와 삭제

- **Copy**는 맵 전체를 새 이름(중복 불가)으로 복제합니다.
- **삭제**하면 맵이 휴지통으로 이동합니다. **설정 → 삭제 예정**에서 **7일 이내** 복구할 수 있고, 이후에는 영구 삭제됩니다.

## 3. 에디터 기본기

맵을 열면 에디터로 들어갑니다. 상단 바에 버전 선택, **새 버전**, **비교**, **실행취소** / **다시실행**, **저장**, 프로세스 라이브러리, **AI 도우미**가 있습니다.

### 캔버스 이동

| 입력 | 동작 |
| --- | --- |
| 마우스 휠 | 세로 이동 (`Shift` + 휠은 가로 이동) |
| `Ctrl` + 휠 | 확대 / 축소 |
| `Space` 누른 채 드래그 | 화면 이동(팬) |
| 빈 캔버스 드래그 | 박스 선택 |
| 우클릭 | 컨텍스트 메뉴(노드·엣지·캔버스) |

### 노드 만들기

캔버스를 우클릭해 도형을 고르거나, 인스펙터의 **Add node**를 사용합니다.

| 유형 | 모양 | 용도 |
| --- | --- | --- |
| **시작(Start)** | 알약 | 흐름의 진입점 — 시작 노드는 하나만 허용 |
| **프로세스(Process)** | 사각형 | 일반 단계·작업 |
| **판단(분기)(Decision)** | 마름모 | 분기 질문 — 출력을 여러 개 가질 수 있는 유일한 노드 |
| **종료(End)** | 알약 | 종료점 — 프로세스당 **대표 끝**은 정확히 하나 |
| **하위프로세스(Subprocess)** | 테두리 박스 | 다른 맵을 참조하는 단계 (5장 참고) |

### 노드 속성

노드를 선택하고 우측 인스펙터에서 편집합니다:

- **제목**과 **설명** — 노드를 더블클릭하거나 `F2`로 즉시 이름을 바꿀 수 있습니다.
- **색상** — 프리셋 견본 또는 커스텀 hex 색상(`#RRGGBB`).
- **BPM 속성** — **담당자**(조직 디렉터리에서 선택), **부서**(담당자에 따라 자동 설정), **시스템**, **소요시간**(예: "2일").

### 노드 연결

- 노드의 핸들을 드래그해 다른 노드에 놓으면 연결됩니다.
- 일반 노드의 출력은 **하나**뿐입니다 — 분기하려면 **판단(분기)** 노드를 쓰세요. 판단 노드에서 나가는 엣지에는 분기 라벨(**Yes** / **No** / **기타**)이 붙습니다.
- 엣지 라벨(분기 조건 등)은 엣지 컨텍스트 메뉴나 `F2`로 편집합니다.
- 엣지별 **선 스타일**: 곡선, 계단, 직선.
- 노드를 다른 노드 가까이 끌면 **드롭존**(**앞** / **그룹** / **뒤**)이 나타나 한 동작으로 흐름에 끼워 넣을 수 있습니다.
- 이미 연결이 있는 노드에 엣지를 놓으면 **사이에 삽입**할지 **기존 유지**할지 묻고, **판단 노드**에 놓으면 **분기 추가** 또는 기존 출력에 **삽입**을 선택합니다.

## 4. 맵 정리하기

### 그룹

- **노드 2개 이상**을 선택하고 `Ctrl+G`(또는 우클릭 → **그룹 만들기**)로 묶습니다.
- 그룹 제목을 더블클릭해 이름을 바꾸고, 제목 바를 드래그해 그룹 전체를 옮기고, **그룹 해제**로 풉니다.
- **그룹 일괄 편집**은 담당자·부서 등 속성을 멤버 전체에 한 번에 설정/해제합니다 — 추가 / 교체 / 건너뛰기 충돌 처리와 전후 요약 제공.

### 정렬과 배치

- **자동 정렬**은 전체 흐름을 자동 배치합니다 — 가로(`Shift+L`, 좌→우) 또는 세로(`Shift+K`, 상→하). 시작→대표 끝 주 경로가 한 직선으로 정렬되고 분기는 옆으로 배치되며, 엣지 출입 방향도 함께 맞춰집니다.
- 2개 이상 선택 시: **왼쪽 정렬** `Alt+W`, **가운데** `Alt+C`, **위쪽 정렬** `Alt+T`, **중간** `Alt+X`.
- 3개 이상 선택 시: **가로 균등 배치** `Alt+R`, **세로 균등 배치** `Alt+V`.

## 5. 하위프로세스 (맵 연결)

**하위프로세스** 노드는 다른 프로세스를 하나의 단계로 끼워 넣습니다 — 복사가 아니라 **참조**입니다.

- **하위프로세스 만들기**(프로세스 노드 우클릭)는 시작 / 작업 / 종료가 준비된 자식 맵을 생성합니다.
- **링크 노드로 추가**는 프로세스 라이브러리에서 기존 맵을 연결합니다. **최신 발행본 자동 추종**을 선택하면 항상 최신 게시 버전을 보여주고, 특정 버전으로 고정할 수도 있습니다. 더 새로운 게시 버전이 생기면 노드에 **최신으로 업데이트**가 표시됩니다.
- 라이브러리 피커에는 **서브프로세스로 지정된 맵만** 나타납니다. 지정은 맵 소유자가 **맵 설정 → 서브프로세스 지정**에서 대표 어트리뷰트(부서 필수, 담당자/시스템/소요시간 선택)와 함께 설정하며, 이 값들은 그 맵을 링크한 모든 노드에 라이브로 표시됩니다.
- **미지정** 맵을 가리키는 링크 노드는 경고 뱃지와 함께 잠기며, 맵이 지정되면 다시 풀립니다.
- **딥 뷰:** 하위프로세스 노드를 더블클릭하면 자식 맵이 브레드크럼과 함께 오버레이로 열립니다 — 임베드된 내용은 **읽기 전용**입니다. `Esc`로 한 단계 위로 나갑니다.
- 연결된 맵의 권한이 없으면 노드에 **No access**가 표시됩니다.

## 6. 노드 찾기와 흐름 따라가기

- **노드 검색** — `/`를 누르고 입력하세요. 한글 **초성 검색**을 지원합니다. 매치된 노드로 점프하며 강조됩니다.
- **아웃라인** — 좌측 레일에 전체 프로세스가 트리로 표시되고, 항목을 클릭하면 해당 노드로 이동합니다.
- **흐름 하이라이트** — 노드를 선택한 뒤 `]`로 강조 경로를 앞으로 늘리고 `[`로 줄이거나 뒤로 확장합니다.
- **흐름 따라 이동** — `Tab` / `Shift+Tab`으로 흐름상 다음 / 이전 노드로 포커스를 옮기며 화면이 따라갑니다.
- **댓글** — 노드마다 댓글 스레드가 있습니다. `Ctrl+Enter`로 전송합니다.

## 7. 저장과 검증

- 편집을 멈추면 약 2초 후 **자동 저장**됩니다. **저장** 버튼은 저장 중… / 저장됨 / 저장 실패 상태를 보여줍니다.
- 저장하지 않고 나가면 경고가 표시됩니다 — 먼저 저장하세요.
- **저장 체크리스트**를 통과해야 저장됩니다: 시작 노드 1개, 대표 끝 1개, 종료 이름 중복 없음, 잘못된 분기 없음(일반 노드 출력 1개).

## 8. 버전과 승인 워크플로

### 버전 모델

하나의 맵은 여러 **버전**(As-Is, To-Be, 자유 라벨)을 가집니다. 새 버전은 기존 버전의 전체 복사본으로 시작합니다. **초안은 한 번에 하나**만 진행할 수 있고, 새 버전은 현재 버전이 게시된 뒤에만 만들 수 있습니다. **버전 번호**는 게시 시점에 부여됩니다.

### 상태

| 상태 | 의미 |
| --- | --- |
| #초안 | 편집 중 — 체크아웃 보유자만 수정 가능 |
| #검토대기 | 승인 요청됨 — 캔버스 읽기 전용 |
| #승인됨 | 모든 결재자가 승인 — 게시 가능 |
| #게시됨 | 모두가 보는 현재 버전 |
| #반려됨 | 결재자가 반려 — 사유와 함께 작성자에게 반환 |
| #만료됨 | 더 새로운 버전이 게시되며 대체됨(최종 상태) |

### 체크아웃 (단일 작성자)

- 초안은 한 번에 한 사람에게 **체크아웃**됩니다. 다른 사람에게는 "{이름} 편집 중 — 읽기 전용" 배너가 표시됩니다.
- **체크아웃 요청**으로 현재 보유자에게 편집권을 요청하면, 보유자가 알림·승인함에서 승인/거절합니다.
- **체크아웃 이전**으로 보유자가 다른 편집자에게 직접 편집권을 넘길 수 있습니다.
- **30분** 동안 활동이 없으면 체크아웃이 자동 해제됩니다.

### 초안에서 게시까지

1. **결재자가 지정되어 있는지** 확인합니다(맵 설정 → 결재자) — 결재자 없이는 승인 요청을 할 수 없습니다.
2. **승인 요청** — 버전이 #검토대기 상태가 되고 잠기며, 모든 결재자에게 알림이 갑니다.
3. 결재자가 **승인** 또는 **반려**합니다(반려는 사유 필수). 승인은 **만장일치** — 모든 결재자가 승인해야 합니다.
4. 승인된 버전을 **게시**합니다. 직전 게시 버전은 자동으로 #만료됨 처리됩니다.
5. **회수**는 검토 대기 버전을 초안으로 되돌립니다(승인 기록 초기화). **재게시**는 게시/만료된 버전에서 새 초안을 만들어 사이클을 다시 시작합니다.

우측 인스펙터 하단의 **승인 대시보드**에 진행 단계(요청 → 검토 → 게시), 결재자별 체크 상태, 가능한 액션이 표시됩니다.

## 9. 버전 비교

에디터 상단 바의 **비교**를 엽니다(게시된 버전이 1개 이상 필요).

- **기준(Base)**과 **대상(Target)** 버전을 고릅니다 — **Swap**으로 서로 바꿉니다.
- 차이가 색으로 강조됩니다: **추가**(초록), **삭제**(빨강), **변경**(노랑). "변경: 필드" 상세와 요약 줄도 제공됩니다.
- **가로 배치** / **세로 배치** 토글로 두 캔버스의 배치를 바꿉니다.
- **변경 목록**은 전체 / 노드 / 엣지로 필터링되고, **속성** 패널은 선택한 노드의 필드를 나란히 보여줍니다.
- **To-Be 적용**으로 대상의 변경을 반영하고, **내보내기**로 비교 결과를 저장합니다.

---

## 10. 맵 설정

에디터에서 **맵 설정**을 엽니다. 탭:

- **상세** — 이름과 설명.
- **협업자** — 개인·팀·유저그룹을 Viewer / Editor로 추가. 편집자 제거·강등은 승인이 필요할 수 있습니다.
- **결재자** — 결재자 명단 관리(버전이 승인 진행 중이면 잠김).
- **공개 범위** — 공개 / 비공개. 변경에는 **승인이 필요**하며 결정 전까지 "승인 대기"로 표시됩니다.
- **버전** — 버전별 워크플로 액션(승인 요청, 게시, 회수 등).
- **승인 대기** / **체크아웃 요청** — 이 맵으로 온 요청을 처리합니다.
- **위험 구역** — **소유권 이전**(나는 편집자로 내려감)과 **맵 삭제**(Owner 전용).

## 11. 공지사항 · 알림·승인 · 피드백

### 공지사항

**공지사항** 탭에 현재 게시 중인 공지가 나옵니다. 전체 / 중요 / 일반 필터, 검색, **읽음 처리**를 지원하며 본문은 마크다운으로 표시됩니다.

### 알림·승인

- **승인 탭** — 내가 처리할 검토 큐: 버전 승인, 체크아웃 이전, 권한·공개 범위 요청. 항목에서 바로 승인/반려(사유 입력)하고 **맵 열기**로 원본으로 이동합니다.
- **알림 탭** — 이벤트 알림이 몇 초 간격으로 갱신됩니다. **모두 읽음** 또는 항목별 읽음 처리.

### 피드백

어느 화면에서든 **피드백**을 클릭해 유형(**버그 / 제안 / 질문 / 기타**)을 고르고 내용을 보내세요 — 현재 화면과 열려 있는 맵이 자동으로 첨부됩니다. 처리 상태와 관리자 답변은 **피드백** 페이지에서 확인합니다.

## 12. 유저그룹

그룹을 쓰면 여러 사람에게 맵 권한을 한 번에 부여할 수 있습니다.

- **그룹 요청**(설정 → 그룹): 이름, 설명, **멤버 2명 이상**, **매니저 1명 이상**(★)을 입력합니다. 관리자가 승인해야 그룹이 활성화됩니다.
- 매니저는 멤버 추가/제거, 매니저 지정, 이름 변경(주 1회), 비활성/재활성, 그룹 삭제를 할 수 있습니다. 삭제된 그룹은 7일간 휴지통에 보관됩니다.
- 내가 속하거나 관리하는 그룹만 보입니다.

## 13. AI 도우미

에디터 상단 바에서 **AI 도우미**를 엽니다(활성화된 경우).

- 자연어 설명으로 플로우차트를 **생성** — 노드·엣지·그룹과 BPM 속성이 조직 디렉터리에 맞춰 채워집니다.
- **증분 편집** — 수정을 요청하면 기존 배치와 색은 유지됩니다. 미리보기 확인 후 **맵에 추가** 또는 취소.
- **분석**("문제 찾기"), **요약**, 단계별 **워크스루**(이전 / 다음 / 자동 재생).
- **사용법 질문** — 이 매뉴얼을 근거로 답합니다.
- **여러 대화** — 대화는 **최대 4개**까지. 대화 바의 목록(카운터 n/4)으로 전환하고, 창 헤더의 **+** 로 새 대화를 시작합니다. 4개인 상태에서 새 대화를 열면 **가장 오래전에 연 대화를 닫을지 확인** 후 닫고 새로 엽니다. 제목은 첫 질문에서 자동으로 만들어집니다.
- **대화 저장과 시간** — 대화는 브라우저에 저장되고 메시지마다 시간이 표시됩니다. 세션당 최근 40개까지 보관하며, 대화 바 아래 **진행바**로 사용량을 확인합니다. 긴 대화는 최근부터 로딩되고 스크롤을 맨 위로 올리면 이전 기록이 이어서 로딩됩니다(로딩 중 기능 팁 노출).
- **입력 한도** — 질문은 한 번에 2000자까지. 입력창 위 오른쪽 **링**이 남은 분량을 보여줍니다(75% 주의, 90% 경고).
- **글자 크기** — 대화 바의 **− T +** 로 채팅 글자 크기를 조절합니다.
- `Ctrl+Enter`로 전송합니다. 읽기 전용 사용자도 분석·워크스루·사용법 질문은 쓸 수 있습니다.

## 15. 자주 묻는 질문

- **편집이 안 돼요.** 다른 사람이 체크아웃 중이거나, 승인 진행 중이거나, 내 역할이 Viewer일 수 있습니다. 캔버스 상단 배너를 확인하세요.
- **승인 요청이 안 돼요.** 먼저 결재자를 1명 이상 지정하세요(맵 설정 → 결재자).
- **새 버전을 못 만들어요.** 현재 초안 사이클을 게시까지 끝내세요 — 초안은 한 번에 하나만 존재할 수 있습니다.
- **맵을 실수로 삭제했어요.** 설정 → 삭제 예정에서 7일 이내에 복구하세요.
- **저장이 안 돼요.** 저장 체크리스트를 확인하세요: 시작 노드 1개, 대표 끝 1개, 종료 이름 중복 없음, 일반 노드 다중 출력 없음.
