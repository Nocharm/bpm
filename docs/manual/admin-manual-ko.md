# Business Process Map — 관리자 매뉴얼

> 이 매뉴얼은 **시스템 관리자(sysadmin)**를 위한 문서입니다. 관리자 콘솔, 운영 업무, 그리고 모든 맵에 걸쳐 sysadmin이 갖는 추가 권한을 다룹니다. **사용자 매뉴얼**(맵 편집 · 사용 안내)을 먼저 읽으세요 — 그 내용 전부가 관리자에게도 적용됩니다.

---

## 1. Sysadmin 역할

### 지정 방식

Sysadmin은 DB에 사용자별로 저장되지 **않습니다** — 백엔드가 기동 시 읽는 **`BPM_SYSADMINS`** 환경변수(로그인 ID 콤마 목록)로 결정됩니다.

```
# backend .env
BPM_SYSADMINS=admin.sys,jane.doe
```

- 인증이 **꺼진** 상태(개발 기본값)에서는 사실상 모든 사용자가 sysadmin입니다.
- `DEV_ENFORCE_PERMISSIONS=true` 또는 실제 Keycloak 인증이 켜지면 `BPM_SYSADMINS`에 등록된 ID만 sysadmin입니다.
- 권한 콘솔에서 해당 사용자에게 **Sysadmin** 태그가 표시되며 *env 관리*로 안내됩니다 — UI에서는 켜고 끌 수 없습니다.

### Sysadmin이 여는 것들

- 모든 맵에서 **Owner로 간주**됩니다 — 어느 맵이든 협업자·결재자·공개 범위·버전·삭제를 관리할 수 있습니다.
- **강제 체크아웃** — 다른 사용자의 활성 편집 잠금을 가져올 수 있는 유일한 역할입니다.
- **설정** 아래 관리자 전용 콘솔: 공지, 임직원, 권한, 데이터베이스, 승인 큐.
- 운영 권한: 유저그룹 승인, 피드백 답변·상태 변경, 전체 휴지통, 매뉴얼 게시.

---

## 2. 관리자 콘솔 지도

모든 관리자 화면은 **설정** 아래에 있습니다. sysadmin으로 로그인하면 좌측 레일에 추가 카테고리가 보입니다:

| 콘솔 | 위치 | 역할 |
| --- | --- | --- |
| **공지(Notices)** | 설정 → 콘텐츠 | 공지 작성·수정·삭제 |
| **매뉴얼(Manual)** | 설정 → 콘텐츠 | 인앱 매뉴얼 편집·게시 (11장 참고) |
| **AI 챗(AI chat)** | 설정 → 콘텐츠 | 대화 보존 상한 설정·채팅 로딩 팁 관리 (12장 참고) |
| **임직원(Employees)** | 설정 → 디렉터리 | 조직 디렉터리 테이블, **AD 전체 동기화** |
| **권한(Permissions)** | 설정 → 권한 | 부서·사용자 탭, sysadmin 태그 |
| **테이블(Tables)** | 설정 → 데이터베이스 | 읽기 전용 DB 브라우저(로그인 기록 포함) |
| **승인 큐(Approval Queue)** | 설정 → 승인 | 전 맵 대상 대기 요청 |
| **대시보드(Dashboard)** | 설정 → 분석 | 실 DB 기반 운영 지표 — 열람 권한을 다른 사람에게 위임 가능(8장) |
| **그룹(Groups)** | 설정 → 그룹 | 그룹 요청 승인, 전체 그룹 조회 |
| **삭제 예정** | 설정 → 휴지통 | 소프트 삭제된 모든 맵·그룹 |

---

## 3. 공지 관리

모든 사용자의 **공지사항** 탭에 표시되는 공지를 관리합니다.

1. **설정 → 콘텐츠 → 공지**로 이동합니다.
2. **새 공지**: 제목, 중요도(**중요** / 일반), 게시 기간(시작–종료 또는 **종료일 없음**), **마크다운** 본문.
3. **게시 시 전체 알림**을 체크하면 모든 사용자의 알림함으로 푸시됩니다.

- 사용자는 게시 기간이 유효한 공지만 봅니다. 관리자 목록에는 전부 표시됩니다.
- 공지 수정·삭제는 즉시 반영됩니다.

> **팁:** 공지 본문은 이 매뉴얼과 같은 마크다운 뷰어로 렌더됩니다 — 헤딩, 표, 코드블록, `#태그` 필이 모두 동작합니다.

---

## 4. 피드백 관리

사용자 피드백(버그 / 제안 / 질문 / 기타)은 발신 시점의 화면과 열려 있던 맵 정보와 함께 **피드백** 페이지로 들어옵니다.

- **상태 변경**과 **답변**은 sysadmin만 할 수 있습니다.
- 항목을 처리 단계에 따라 이동시키세요. **완료** 처리하면 완료 시각이 기록되고 **추가 답변이 잠깁니다**.
- 사용자는 자기 제출 건을 지켜보고 있으므로, 상태 변경과 답변이 즉시 그들에게 보입니다.

---

## 5. 유저그룹 관리

그룹 생성은 요청 기반입니다. 누구나 그룹을 요청할 수 있지만, sysadmin이 승인해야 **활성** 상태가 됩니다.

- 대기 중인 요청은 **승인 큐**(및 그룹 대기 목록)에 나타납니다. 이름, 멤버(최소 2명), 매니저(최소 1명)를 확인하고 승인/반려하세요.
- Sysadmin은 비활성·삭제된 그룹을 포함해 **모든 그룹**을 볼 수 있습니다. 일반 사용자는 자기가 속한 그룹만 봅니다.
- 삭제된 그룹은 **7일간** 휴지통에 있다가 영구 삭제됩니다. **설정 → 휴지통**에서 복구하세요.

---

## 6. 전체 승인 큐

**설정 → 승인 → 승인 큐**는 전 맵의 대기 요청을 한곳에 모읍니다:

- **그룹 생성** 요청.
- **권한 강등** 요청(편집자 제거·강등).
- **공개 범위 변경** 요청(공개 ↔ 비공개).

각 항목에 요청자와 맥락이 표시되며 승인 / 반려(사유 필수)로 처리합니다. 맵 단위 요청은 해당 맵의 결재자도 처리할 수 있습니다 — 이 큐는 놓친 요청이 없는지 확인하는 전체 뷰입니다.

---

## 7. 디렉터리와 임직원

**설정 → 디렉터리 → 임직원**은 담당자·협업자·결재자 선택에 쓰이는 조직 디렉터리를 보여줍니다.

- 테이블에 조직 단계와 사용자별 sysadmin 플래그가 포함됩니다.
- **AD 전체 동기화**로 Active Directory에서 디렉터리를 갱신합니다. 입사·이동·퇴사가 있으면 실행하세요.
- 에디터의 담당자 선택기는 이 디렉터리를 참조합니다 — 디렉터리가 오래되면 선택기에서 사람이 빠집니다.

AD가 채우지 못하는 한글 이름·부서 정보는 임포트 도구로 보완합니다:

- **한글 이름 임포트** — 직원 한글 이름을 CSV로 올려 채웁니다. 스키마와 현재 데이터(누락분 / 샘플 / 랜덤 50 / 전체)를 내려받아 준비하고, 기존 값과 충돌하면 **건너뛰기 / 덮어쓰기**를 고릅니다. 피커·멤버 카드의 한글 이름 표시에 쓰입니다.
- **부서 정보 임포트** — 부서의 한글 이름과 부서장을 올립니다. 피커의 한글 부서 검색과 부서장 체인 표시에 쓰입니다.
- **부서 재지정** — 직원의 소속 부서를 다른 부서로 옮깁니다. **부서 테이블**에서 조직 부서 목록을 확인할 수 있습니다.

---

## 8. 데이터베이스 뷰어와 운영 대시보드

**설정 → 데이터베이스 → 테이블**은 서버 측 페이징·정렬·필터를 지원하는 읽기 전용 DB 브라우저입니다.

- 데이터 확인 용도로만 사용하세요 — 쓰기는 불가능합니다.
- **로그인 기록**은 `login_records` 테이블에 있습니다: 사용자당 하루 한 행, 그날 첫 인증 시 기록됩니다. "누가 언제 썼는지"의 감사 뷰입니다.

### 대시보드 (설정 → 분석)

실 DB 기준 운영 지표를 한눈에 보여줍니다:

- **운영(Operations)** — 전체 맵·게시본·진행 중·휴지통 수, 열린 댓글, 미읽음 알림, 체크아웃 이전 요청.
- **버전 상태(Version status)** — 초안·검토대기·승인·게시·만료 버전 분포.
- **부서별 도입률(Adoption by department)** — 맵을 보유한 부서 비율. 분모로 셀 부서는 **커버리지(Coverage)** 사이드바에서 관리자가 고릅니다.
- **로그인·활동**, **누적 증가**(맵·버전 생성 추이) — 기간 필터(7일 / 1개월 / 3개월 / 사용자 지정)로 시계열을 조정합니다(현재 상태 지표는 기간 필터의 영향을 받지 않습니다).
- **최근 버전 이벤트**, **AI 사용량**(7일·30일 호출 수, 상위 맵).

**대시보드 열람 권한**은 sysadmin이 위임할 수 있습니다 — 대시보드 우측 **접근(Access)** 사이드바에서 사람·부서·유저그룹에 **권한 부여**하면 sysadmin이 아니어도 대시보드를 열 수 있습니다(sysadmin은 항상 열람 가능). 권한이 하나도 없으면 sysadmin만 봅니다.

---

## 9. 휴지통과 복구

맵·그룹 삭제는 **소프트 삭제**입니다 — 휴지통으로 이동하고 **7일** 후 영구 삭제됩니다.

- 소유자는 **설정 → 삭제 예정**에서 자기 맵만 보지만, sysadmin은 **전원의 것**을 봅니다.
- **복구**하면 맵이 버전·노드·권한 그대로 돌아옵니다.
- 7일이 지나면 영구 삭제되어 되돌릴 수 없습니다.

---

## 10. 버전 워크플로 — 관리자 권한

사용자 매뉴얼의 워크플로 내용에 더해:

- **강제 체크아웃** — 보유자가 부재할 때 활성 편집 잠금을 가져옵니다. 이전 보유자의 미저장 작업 맥락이 사라지므로 신중히 사용하세요. 유휴 잠금은 이미 30분 후 자동 해제됩니다.
- **어디서든 결정** — 실질 Owner이므로 어느 맵에서든 승인 요청·게시를 하고 체크아웃 요청·이전을 처리할 수 있습니다. 단, **회수(Withdraw)**는 예외입니다 — 검토대기·승인됨 버전은 **제출자만** 회수할 수 있고, 반려된 버전만 맵 오너·sysadmin이 회수할 수 있습니다.
- **결재자 재지정** — 활성 결재자가 없는 맵(예: 유일한 결재자 퇴사)은 맵 설정 → 결재자의 강제 재지정 플로로 새 결재자를 지정합니다.
- 게시 규칙 유의: 게시하면 직전 게시 버전이 #만료됨 처리되며, 이는 승인에 다시 들어갈 수 없는 최종 상태입니다. 다시 진행하려면 **재게시**로 새 초안을 만드세요.

## 11. 이 매뉴얼 게시하기

**/manual**은 여러 문서를 담을 수 있습니다(공지사항처럼). **설정 → 콘텐츠 → 매뉴얼**에서 관리합니다:

1. **문서 목록**에서 기존 문서를 클릭해 수정하거나 **새 문서**로 추가합니다. 행의 휴지통 버튼으로 삭제합니다.
2. 포맷(**마크다운**/**HTML** — HTML은 렌더 전에 sanitize)과 **언어(한국어/English)**를 고릅니다. 뷰어에는 상단 한/영 토글에 맞는 언어의 문서만 목록에 뜹니다.
3. 편집기에 직접 쓰거나, **.md 업로드**로 파일을 불러오거나, **배포본 불러오기**로 빌드에 포함된 원문에서 시작합니다.
4. **미리보기**는 사용자가 보게 될 화면 그대로 렌더하고, **게시**하면 즉시 /manual에 반영됩니다.

- 목록 제목은 저장 시 본문 **첫 헤딩에서 자동 추출**됩니다.
- 한/영 문서를 짝지어 같은 순서로 올리면, 뷰어에서 언어를 전환해도 같은 순번의 문서가 열립니다.
- 등록된 문서가 없으면 뷰어는 빌드에 포함된 기본 매뉴얼을 보여줍니다.
- **번들 fallback** — 한 번도 게시된 적이 없으면 빌드에 포함된 `backend/app/manual.md`가 제공되고, 뷰어에 갱신 시각 대신 **배포 포함본** 뱃지가 표시됩니다.
- 같은 게시를 sysadmin API로도 할 수 있습니다:

```
PUT /api/manual
{ "format": "markdown", "content": "<마크다운 전문>" }
```

뷰어는 `##`·`###` 헤딩으로 목차를 만들므로 문서를 그 구조로 작성하세요. 렌더러는 헤딩, 평면 리스트, 표, 펜스 코드블록(호버 복사), 인라인 코드(클릭 복사), 인용, 링크, 굵게/기울임, `#단어` 필을 지원합니다 — 이미지와 raw HTML은 렌더되지 않습니다.

## 12. AI 챗 설정

**설정 → 콘텐츠 → AI 챗** (sysadmin 전용). 재배포 없이 즉시 적용됩니다.

- **대화 저장**: AI 챗 대화는 항상 서버에 저장됩니다(사용자·맵 단위로 귀속되며 본인만 조회할 수 있습니다) — 켜고 끌 수 있는 옵션이 아닙니다. `ai_chat_sessions`(대화)·`ai_chat_messages`(메시지) 테이블에 쌓이며 테이블 뷰어에서 조회할 수 있습니다.
- **보존 상한**: 맵당 대화 수·대화당 메시지 수·마지막 활동 후 보관 일수를 관리자가 조정합니다. 초과분은 오래된 것부터 자동 정리됩니다.

| 키 | 기본값 | 범위 |
| --- | --- | --- |
| `ai_chat_max_sessions_per_map` | 20 | 1–200 |
| `ai_chat_max_messages_per_session` | 200 | 10–2000 |
| `ai_chat_retention_days` | 180 | 7–3650 |

- **채팅 로딩 팁**: 이전 대화를 불러오는 동안 채팅에 노출되는 기능 팁을 관리합니다. 한 줄당 팁 1개(팁당 200자, 최대 50개). **모두 지우고 저장하면 기본 20종으로 복원**됩니다.

## 13. 설정 레퍼런스

| 변수 | 위치 | 효과 |
| --- | --- | --- |
| `BPM_SYSADMINS` | backend `.env` | sysadmin으로 지정할 로그인 ID 콤마 목록 |
| `AUTH_ENABLED` | backend `.env` | Keycloak JWT 검증 활성화 |
| `NEXT_PUBLIC_AUTH_ENABLED` | frontend env | UI의 Keycloak 로그인 플로 활성화 |
| `DEV_ENFORCE_PERMISSIONS` | backend `.env` | 인증 없이도 로컬에서 RBAC 강제 |
| `MANUAL_URL` | `.env` (compose) | 에디터 툴바의 매뉴얼 사이트 버튼 — 비우면 숨김 |

- 환경변수 변경은 백엔드 재기동이 필요합니다(`--reload`는 `.env`를 다시 읽지 않습니다).
- Keycloak 주소 등 배포별 값은 전부 `.env`로 관리합니다 — 하드코딩 금지.
