# Business Process Map — 맵 편집

> Business Process Map(BPM)에서 **프로세스맵을 그리고 편집하는 방법**을 다룹니다 — 노드·연결·그룹·회당 파라미터·하위프로세스·저장·가져오기/내보내기·AI 도우미까지. 로그인·맵 목록·버전 승인·설정 등 편집 외의 사용법은 **사용 안내** 매뉴얼을 함께 보세요.

---

## 1. 에디터 둘러보기

맵을 열면 에디터로 들어갑니다. 상단 바에 버전 선택, **새 버전**, **비교**, **실행취소** / **다시실행**, **저장**, 프로세스 라이브러리, **가져오기(CSV)**, **AI 도우미**가 있습니다. 우측 인스펙터에서 선택한 노드의 속성을 편집하고, 좌측 사이드바에 아웃라인 트리와 단축키 카드가 있습니다.

### 캔버스 이동

| 입력 | 동작 |
| --- | --- |
| 마우스 휠 | 세로 이동 (`Shift` + 휠은 가로 이동) |
| `Ctrl` + 휠 | 확대 / 축소 |
| `Space` 누른 채 드래그 | 화면 이동(팬) |
| 빈 캔버스 드래그 | 박스 선택 |
| 우클릭 | 컨텍스트 메뉴(노드·엣지·캔버스) |

---

## 2. 노드와 연결

### 노드 만들기

캔버스를 우클릭해 도형을 고르거나, 인스펙터의 **Add node**를 사용합니다.

| 유형 | 모양 | 용도 |
| --- | --- | --- |
| **시작(Start)** | 알약 | 흐름의 진입점 — 시작 노드는 하나만 허용 |
| **프로세스(Process)** | 사각형 | 일반 단계·작업 |
| **판단(분기)(Decision)** | 마름모 | 분기 질문 — 출력을 여러 개 가질 수 있는 유일한 노드 |
| **종료(End)** | 알약 | 종료점 — 프로세스당 **대표 끝**은 정확히 하나 |
| **하위프로세스(Subprocess)** | 테두리 박스 | 다른 맵을 참조하는 단계 (5장 참고) |

### 노드 속성

노드를 선택하고 우측 인스펙터에서 편집합니다:

- **제목**과 **설명** — 노드를 더블클릭하거나 `F2`로 즉시 이름을 바꿀 수 있습니다.
- **색상** — 프리셋 견본 또는 커스텀 hex 색상(`#RRGGBB`).
- **BPM 속성** — **담당자**(조직 디렉터리에서 선택), **부서**(담당자에 따라 자동 설정), **시스템**, 그리고 **회당 파라미터**(3장 참고).
- **링크(URL)** — 노드에 외부 문서·시스템 링크를 달면 캔버스에 배지가 뜨고, 클릭해 미리보기·새 탭 열기를 할 수 있습니다. 표시 이름(라벨)을 따로 지정할 수 있습니다.

### 노드 연결

- 노드의 핸들을 드래그해 다른 노드에 놓으면 연결됩니다.
- 일반 노드의 출력은 **하나**뿐입니다 — 분기하려면 **판단(분기)** 노드를 쓰세요. 판단 노드에서 나가는 엣지에는 분기 라벨(**Yes** / **No** / **기타**)이 붙습니다.
- 엣지 라벨(분기 조건 등)은 엣지 컨텍스트 메뉴나 `F2`로 편집합니다.
- 엣지별 **선 스타일**: 곡선, 계단, 직선.
- 노드를 다른 노드 가까이 끌면 **드롭존**(**앞** / **그룹** / **뒤**)이 나타나 한 동작으로 흐름에 끼워 넣을 수 있습니다.
- 이미 연결이 있는 노드에 엣지를 놓으면 **사이에 삽입**할지 **기존 유지**할지 묻고, **판단 노드**에 놓으면 **분기 추가** 또는 기존 출력에 **삽입**을 선택합니다.

## 3. 회당 파라미터

각 노드는 프로세스를 **한 번 수행할 때**의 비용·소요를 기록하는 6개 파라미터를 가집니다. 인스펙터의 파라미터 섹션(기본 접힘)에서 편집합니다. **시작·종료** 노드는 파라미터가 없습니다.

| 파라미터 | 라벨 | 입력 형식 | 캔버스 표시 |
| --- | --- | --- | --- |
| **소요시간** | 회당 소요시간(h) | `시.분` — 소수부는 **분** | `1h30m` |
| **비용(원)** | 회당 추가비용(원) | 숫자 | `₩1,250,000` |
| **비용($)** | 회당 추가비용($) | 숫자 | `$1,200` |
| **투입인원** | 회당 투입인원 | 숫자 | 그대로 |
| **연간 건수** | 연간 건수 | 숫자 | 그대로 |
| **FTE** | FTE | 숫자 | 그대로 |

### 소요시간 표기 (시.분)

소요시간은 **시간과 분**으로 적습니다 — 소수부는 십진수가 **아니라 분**입니다.

- `2` → 2시간, `0.30` → 30분, `1.30` → 1시간 30분.
- 분이 60을 넘으면 시간으로 올라갑니다(`0.90` → 1시간 30분).
- 편집 중에는 입력한 `1.30` 형태로 보이고, 그 외 모든 화면에서는 `1h30m`으로 표시됩니다.

### 비용은 한 통화만

**원(₩)과 달러($)는 동시에 입력할 수 없습니다.** 한쪽에 값을 넣으면 반대쪽 입력이 잠깁니다. 통화를 바꾸려면 먼저 채워진 쪽을 비우세요. 비용에는 천단위 콤마가 자동으로 붙고, `1,250,000`처럼 콤마를 넣어 붙여넣어도 됩니다.

### 하위프로세스 노드의 파라미터

하위프로세스 노드는 **연간 건수**와 **FTE**만 직접 입력합니다. 나머지 4개(소요시간·비용·투입인원)는 **링크한 맵의 지정값을 읽기전용으로 상속**하며, 부모 맵에서는 바꿀 수 없습니다(5장 참고).

### 합계 미리보기 (Σ)

맵을 하위프로세스로 지정할 때(사용 안내 매뉴얼의 맵 설정 참고), 지정 폼이 그 맵의 게시본 파라미터를 **합산해 미리보기**로 제안합니다 — 소요시간·비용은 **합**, 투입인원은 값이 있는 일반 노드의 **평균**입니다.

## 4. 맵 정리하기

### 그룹

- **노드 2개 이상**을 선택하고 `Ctrl+G`(또는 우클릭 → **그룹 만들기**)로 묶습니다.
- 그룹 제목을 더블클릭해 이름을 바꾸고, 제목 바를 드래그해 그룹 전체를 옮기고, **그룹 해제**로 풉니다.
- **그룹 일괄 편집**은 담당자·부서 등 속성을 멤버 전체에 한 번에 설정/해제합니다 — 추가 / 교체 / 건너뛰기 충돌 처리와 전후 요약 제공.

### 정렬과 배치

- **자동 정렬**은 전체 흐름을 자동 배치합니다 — 가로(`Shift+L`, 좌→우) 또는 세로(`Shift+K`, 상→하). 시작→대표 끝 주 경로가 한 직선으로 정렬되고 분기는 옆으로 배치되며, 엣지 출입 방향도 함께 맞춰집니다.
- 2개 이상 선택 시: **왼쪽 정렬** `Alt+W`, **가운데** `Alt+C`, **위쪽 정렬** `Alt+T`, **중간** `Alt+X`.
- 3개 이상 선택 시: **가로 균등 배치** `Alt+R`, **세로 균등 배치** `Alt+V`.

## 5. 하위프로세스 (맵 연결)

**하위프로세스** 노드는 다른 프로세스를 하나의 단계로 끼워 넣습니다 — 복사가 아니라 **참조**입니다.

- **하위프로세스 만들기**(프로세스 노드 우클릭)는 시작 / 작업 / 종료가 준비된 자식 맵을 생성합니다.
- **링크 노드로 추가**는 프로세스 라이브러리에서 기존 맵을 연결합니다. **최신 발행본 자동 추종**을 선택하면 항상 최신 게시 버전을 보여주고, 특정 버전으로 고정할 수도 있습니다. 더 새로운 게시 버전이 생기면 노드에 **최신으로 업데이트**가 표시됩니다.
- 라이브러리 피커에는 **서브프로세스로 지정된 맵만** 나타납니다. 지정은 맵 소유자가 **맵 설정 → 서브프로세스 지정**에서 대표 어트리뷰트(부서 필수, 담당자/시스템/소요시간·비용·투입인원 선택)와 함께 설정하며, 이 값들은 그 맵을 링크한 모든 노드에 라이브로 표시됩니다.
- **미지정** 맵을 가리키는 링크 노드는 경고 뱃지와 함께 잠기며, 맵이 지정되면 다시 풀립니다.
- **딥 뷰:** 하위프로세스 노드를 더블클릭하면 자식 맵이 브레드크럼과 함께 오버레이로 열립니다 — 임베드된 내용은 **읽기 전용**입니다. `Esc`로 한 단계 위로 나갑니다.
- 연결된 맵의 권한이 없으면 노드에 **No access**가 표시됩니다.

## 6. 노드 찾기와 흐름 따라가기

- **노드 검색** — `/`를 누르고 입력하세요. 한글 **초성 검색**을 지원합니다. 매치된 노드로 점프하며 강조됩니다.
- **아웃라인** — 좌측 레일에 전체 프로세스가 트리로 표시되고, 항목을 클릭하면 해당 노드로 이동합니다.
- **흐름 하이라이트** — 노드를 선택한 뒤 `]`로 강조 경로를 앞으로 늘리고 `[`로 줄이거나 뒤로 확장합니다.
- **흐름 따라 이동** — `Tab` / `Shift+Tab`으로 흐름상 다음 / 이전 노드로 포커스를 옮기며 화면이 따라갑니다.
- **댓글** — 노드마다 댓글 스레드가 있습니다. `Ctrl+Enter`로 전송합니다. 읽기 전용 사용자도 댓글은 남길 수 있습니다.

## 7. 저장과 검증

- 편집을 멈추면 약 2초 후 **자동 저장**됩니다. **저장** 버튼은 저장 중… / 저장됨 / 저장 실패 상태를 보여줍니다.
- 저장하지 않고 나가면 경고가 표시됩니다 — 먼저 저장하세요.
- **저장 체크리스트**를 통과해야 저장됩니다:
  - 시작 노드 **1개**
  - 대표 끝(종료) **1개**
  - 종료 이름 **중복 없음**
  - 잘못된 분기 없음 — 일반 노드의 출력은 **1개**뿐(여러 출력은 판단 노드만)
- 편집은 초안(#초안 / #반려됨)이고 내가 **체크아웃**을 쥐고 있을 때만 가능합니다. 다른 사람이 편집 중이거나 승인 진행 중이면 캔버스가 읽기 전용으로 잠깁니다(버전·승인은 사용 안내 매뉴얼 참고).

## 8. 가져오기 (CSV · AI)

표나 문서로 정리된 프로세스를 붙여넣어 맵을 채울 수 있습니다.

### CSV 가져오기

- 상단 바 **가져오기(Import CSV)**로 엽니다. **템플릿 다운로드**로 빈 양식을 받아 채운 뒤 올리세요.
- CSV는 **14개 열**을 씁니다: `name`(필수), `description`, `assignee`, `department`, `system`, `duration`, `cost_krw`, `cost_usd`, `headcount`, `annual_count`, `fte`, `url`, `url_label`, `next`(선후 연결 대상).
- 가져오기는 **제목(name)으로 병합**합니다 — 같은 제목의 기존 노드는 색·댓글·그룹을 그대로 두고 값만 갱신하며, **빈 칸은 기존 값을 유지**합니다. CSV에 없는 새 제목은 노드로 추가됩니다.
- **담당자**는 이름으로 적으면 조직 디렉터리와 매칭됩니다. **비용은 원/달러 중 한쪽만**, **소요시간은 시.분** 규칙을 그대로 따릅니다(3장). 하위프로세스 노드가 상속하는 4필드는 CSV로 넣어도 무시됩니다.
- 미리보기 탭에서 **추가 / 매칭 / 삭제** 요약과 경고를 확인한 뒤 적용합니다.

> **외부 AI로 CSV 만들기:** 가져오기 창의 **다른 AI에게 물어보기**로 프롬프트를 복사해, 문서와 함께 외부 AI(ChatGPT 등)에 붙여넣고 받은 CSV를 다시 여기에 붙여넣을 수 있습니다.

### CSV로 새 맵 만들기

맵 목록(홈)의 **New map** 옆 드롭다운에서 **Create from CSV**를 고르면, CSV를 드롭해 바로 새 맵을 생성합니다(사용 안내 매뉴얼의 맵 목록 참고).

### AI 제안 병합

AI 도우미가 만든 순서도 제안도 같은 방식으로 **제목 기준 병합**됩니다(id 보존) — 자세한 내용은 10장.

## 9. 내보내기 (PNG · Excel · CSV · Word)

우측 인스펙터의 내보내기 버튼(또는 우클릭 메뉴)에서 현재 맵을 파일로 저장합니다.

| 형식 | 내용 |
| --- | --- |
| **PNG** | 현재 캔버스를 2배 해상도 이미지로 — 모든 연결선이 검은 실선으로 출력. `Ctrl+Shift+E`. |
| **Excel** | 노드 표(담당자·부서·시스템·회당 파라미터 포함). 비용은 통화별 열과 숫자 서식으로 저장돼 스프레드시트에서 계산할 수 있습니다. |
| **CSV** | 가져오기와 같은 14열 표 — 내보낸 CSV를 고쳐 다시 가져오는 왕복이 가능합니다. |
| **Word** | 도형 순서도가 그려진 `.docx` 문서 — 노드 링크(URL)는 하이퍼링크로 포함됩니다. |

> CSV·Excel은 캔버스 좌표·그룹처럼 표로 표현되지 않는 구조가 있으면 경고와 함께 내보냅니다. 아주 큰 맵은 행 상한에서 잘릴 수 있습니다.

## 10. AI 도우미

에디터 상단 바에서 **AI 도우미**를 엽니다(서버에서 AI가 활성화된 경우에만 보입니다).

- 자연어 설명으로 플로우차트를 **생성** — 노드·엣지·그룹과 BPM 속성(담당자·부서·시스템·회당 파라미터)이 조직 디렉터리에 맞춰 채워집니다.
- **증분 편집** — 수정을 요청하면 기존 배치와 색·담당자·그룹은 유지됩니다. 노드 추가/삭제, 연결/연결 끊기, 두 노드 사이 삽입, 분기 라벨 변경, 노드 설명·링크(URL) 설정을 지원합니다. 미리보기 확인 후 **맵에 추가(Apply)** 또는 취소(Discard).
- **분석**("문제 찾기"), **요약**, 단계별 **워크스루**(이전 / 다음 / 자동 재생).
- **사용법 질문** — 이 매뉴얼을 근거로 답합니다. 매뉴얼 밖 내용은 모른다고 답합니다.
- **여러 대화** — 대화는 서버에 저장되어 어느 기기에서든 이어집니다. 대화 바의 목록에서 이전 대화를 열거나 창 헤더의 **+** 아이콘으로 새 대화를 시작합니다. 대화 제목은 첫 질문에서 자동으로 만들어지고, 목록에서 대화를 삭제할 수 있습니다.
- **다른 맵의 대화** — 대화 목록의 "다른 맵 대화"에서 다른 맵에서 나눈 대화를 읽기 전용으로 열람할 수 있습니다. 이어서 입력하려면 "이 맵 열기" 버튼으로 해당 맵으로 이동합니다.
- **대화 저장과 시간** — 대화는 맵 단위로 서버에 저장되고 메시지마다 시간이 표시됩니다. 보관량은 관리자가 정한 상한(기본: 맵당 대화 20개·대화당 메시지 200개·마지막 활동 후 180일)까지이며, 초과분은 오래된 것부터 자동 정리됩니다. 긴 대화는 최근부터 로딩되고, 스크롤을 맨 위로 올리면 이전 기록이 이어서 로딩됩니다(로딩 중 기능 팁 노출).
- **입력 한도** — 질문은 한 번에 2000자까지. 입력창 위 오른쪽 **링**이 남은 분량을 보여줍니다(75% 주의, 90% 경고).
- **글자 크기** — 대화 바의 **− T +** 로 채팅 글자 크기를 조절합니다.
- `Ctrl+Enter`로 전송합니다. 생성·편집은 편집 권한이 필요하고, 분석·워크스루·사용법 질문은 읽기 전용 사용자도 쓸 수 있습니다.
