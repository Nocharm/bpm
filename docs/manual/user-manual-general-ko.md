# Business Process Map — 사용 안내

> Business Process Map(BPM)은 업무 프로세스를 플로우차트로 그리고, As-Is / To-Be **버전**으로 관리하며, **승인 워크플로**로 확정하는 웹 서비스입니다.

이 매뉴얼은 로그인부터 맵 관리·버전 승인·설정·협업까지 **편집 외의 전반적인 사용법**을 다룹니다. 캔버스에서 노드를 그리고 편집하는 방법은 **맵 편집** 매뉴얼을 함께 보세요. 시스템 관리자는 **관리자 매뉴얼**도 읽어 주세요.

---

## 1. 시작하기

### 로그인

1. 앱에 접속해 **Sign in with Keycloak**을 클릭하면 사내 SSO 계정으로 로그인됩니다. 한 번 로그인해 두면 다음 방문부터는 자동으로 로그인되고, 처음 열려던 화면(딥링크)으로 이어집니다.
2. 개발 환경에서는 **Sign in with a test account**로 테스트 사용자를 골라 로그인할 수 있습니다.

### 화면 한눈에 보기

| 화면 | 위치 | 역할 |
| --- | --- | --- |
| **맵 목록(Maps)** | 상단 탭 | 전체 프로세스맵 — 검색·생성·열기 |
| **공지사항(Notices)** | 상단 탭 | 관리자가 게시한 공지 |
| **알림·승인(Inbox)** | 상단 탭 | 내가 처리할 승인 요청과 알림 |
| **사용 매뉴얼(Manual)** | 상단 바 | 매뉴얼 문서들(한/영) — 제목 드롭다운으로 선택, 목차와 본문 검색 지원 |
| **피드백(Feedback)** | 상단 바 | 버그·제안을 관리자에게 전송 |
| **설정(Settings)** | 상단 바 | 내 그룹, 삭제 예정(휴지통), 관리자 콘솔 |

> **팁:** 맵 목록과 이 매뉴얼에서 `/` 키를 누르면 바로 검색창으로 이동합니다.

### 맵에서의 역할

| 역할 | 할 수 있는 일 |
| --- | --- |
| **Owner** | 전부 — 편집, 협업자·결재자·공개 범위 관리, 삭제 |
| **Editor** | 맵과 초안 버전 편집 |
| **Viewer** | 읽기 전용 — 열람, 비교, 댓글 |
| **결재자(Approver)** | 승인 요청된 버전의 검토·승인/반려 |

맵마다 **오우닝 부서(Owning department)**가 있으며, 그 부서에 속한 사람은 자동으로 **Editor** 권한을 갖습니다(2장 맵 만들기 · 5장 맵 설정).

공개 범위가 **공개(Public)**인 맵은 누구나 볼 수 있고, **비공개(Private)** 맵은 협업자와 결재자만 볼 수 있습니다. 볼 권한이 없는 비공개 맵으로 이동하면 접근 불가 안내가 표시됩니다.

## 2. 맵 목록 (홈)

### 맵 찾기

- **최근 열어본** 밴드에 마지막으로 작업한 맵이 표시됩니다 — **Show more** / **Collapse**로 펼치고 접습니다.
- **All maps**에는 내가 볼 수 있는 모든 맵이 나옵니다. `/` 키 또는 검색창(**Search maps**)으로 이름을 필터링하세요.
- 필터 칩으로 **상태**(초안 / 검토 대기 / 승인됨 / 반려됨 / 게시됨)와 내 **역할** 기준으로 좁힐 수 있습니다. **Clear filters**로 초기화합니다.
- 카드를 클릭하면 **상세 패널**이 열립니다: 소유자, 노드 수, 열람 가능 멤버(개인 / 팀 / 유저그룹), 버전별 이력(생성·승인요청·승인·반려·게시·회수).

### 맵 만들기

1. **New map**을 클릭합니다.
2. 이름과 설명을 입력하고 **공개 범위**(공개 / 비공개)를 선택합니다.
3. **오우닝 부서**를 지정합니다(필수) — 실제 조직 부서여야 하며, 그 부서에 속한 사람은 이 맵의 Editor 권한을 항상 갖습니다.
4. 필요하면 **초기 협업자**를 추가합니다 — 개인·팀·유저그룹을 Viewer 또는 Editor로.
5. **결재자를 1명 이상** 지정합니다 — 결재자 없이는 맵을 만들 수 없습니다.

> **CSV로 새 맵 만들기:** **New map** 옆 드롭다운에서 **Create from CSV**를 고르면 CSV 파일을 드롭해 노드가 채워진 맵을 바로 만듭니다. CSV 형식과 규칙은 **맵 편집** 매뉴얼의 "가져오기"를 참고하세요.

### 복사와 삭제

- **Copy**는 맵 전체를 새 이름(중복 불가)으로 복제합니다.
- **삭제**하면 맵이 휴지통으로 이동합니다. **설정 → 삭제 예정**에서 **7일 이내** 복구할 수 있고, 이후에는 영구 삭제됩니다.

## 3. 버전과 승인 워크플로

### 버전 모델

하나의 맵은 여러 **버전**(As-Is, To-Be, 자유 라벨)을 가집니다. 새 버전은 기존 버전의 전체 복사본으로 시작합니다. **초안은 한 번에 하나**만 진행할 수 있고, 새 버전은 현재 버전이 게시된 뒤에만 만들 수 있습니다. **버전 번호**는 게시 시점에 부여됩니다.

### 상태

| 상태 | 의미 |
| --- | --- |
| #초안 | 편집 중 — 체크아웃 보유자만 수정 가능 |
| #검토대기 | 승인 요청됨 — 캔버스 읽기 전용 |
| #승인됨 | 모든 결재자가 승인 — 게시 가능 |
| #게시됨 | 모두가 보는 현재 버전 |
| #반려됨 | 결재자가 반려 — 사유와 함께 작성자에게 반환 |
| #만료됨 | 더 새로운 버전이 게시되며 대체됨(최종 상태) |

### 체크아웃 (단일 작성자)

- 초안은 한 번에 한 사람에게 **체크아웃**됩니다. 다른 사람에게는 "{이름} 편집 중 — 읽기 전용" 배너가 표시됩니다.
- **체크아웃 요청**으로 현재 보유자에게 편집권을 요청하면, 보유자가 알림·승인함에서 승인/거절합니다.
- **체크아웃 이전**으로 보유자가 다른 편집자에게 직접 편집권을 넘길 수 있습니다.
- **30분** 동안 활동이 없으면 체크아웃이 자동 해제됩니다.

### 초안에서 게시까지

1. **결재자가 지정되어 있는지** 확인합니다(맵 설정 → 결재자) — 결재자 없이는 승인 요청을 할 수 없습니다.
2. **승인 요청** — 버전이 #검토대기 상태가 되고 잠기며, 모든 결재자에게 알림이 갑니다.
3. 결재자가 **승인** 또는 **반려**합니다(반려는 사유 필수). 승인은 **만장일치** — 모든 결재자가 승인해야 합니다.
4. 승인된 버전을 **게시**합니다. 직전 게시 버전은 자동으로 #만료됨 처리됩니다.
5. **회수**는 검토 대기(Pending)·승인됨(Approved) 버전을 초안으로 되돌립니다(승인 기록 초기화) — **제출자만** 할 수 있습니다. 반려된(Rejected) 버전은 제출자 외에 맵 오너·sysadmin도 회수할 수 있으며, 회수한 사람이 편집권(체크아웃)을 갖습니다. **재게시**는 게시/만료된 버전에서 새 초안을 만들어 사이클을 다시 시작합니다.

우측 인스펙터 하단의 **승인 대시보드**에 진행 단계(요청 → 검토 → 게시), 결재자별 체크 상태, 가능한 액션이 표시됩니다.

## 4. 버전 비교

에디터 상단 바의 **비교**를 엽니다(게시된 버전이 1개 이상 필요).

- **기준(Base)**과 **대상(Target)** 버전을 고릅니다 — **Swap**으로 서로 바꿉니다.
- 차이가 색으로 강조됩니다: **추가**(초록), **삭제**(빨강), **변경**(노랑). "변경: 필드" 상세와 요약 줄도 제공됩니다.
- **가로 배치** / **세로 배치** 토글로 두 캔버스의 배치를 바꿉니다.
- **변경 목록**은 전체 / 노드 / 엣지로 필터링되고, **속성** 패널은 선택한 노드의 필드(회당 파라미터 포함)를 나란히 보여줍니다.
- **To-Be 적용**으로 대상의 변경을 반영하고, **내보내기**로 비교 결과를 저장합니다.

## 5. 맵 설정

에디터에서 **맵 설정**을 엽니다. 탭:

- **상세** — 이름과 설명.
- **오우닝 부서** — 맵의 오우닝 부서 지정/변경(Owner·sysadmin 전용). 부서원은 자동으로 Editor 권한을 가지며, 부서를 바꾸면 파생 Editor도 새 부서를 따라갑니다.
- **협업자** — 개인·팀·유저그룹을 Viewer / Editor로 추가. 편집자 제거·강등은 승인이 필요할 수 있습니다.
- **결재자** — 결재자 명단 관리(버전이 승인 진행 중이면 잠김).
- **공개 범위** — 공개 / 비공개. 변경에는 **승인이 필요**하며 결정 전까지 "승인 대기"로 표시됩니다.
- **서브프로세스 지정** — 이 맵을 다른 맵의 하위프로세스로 쓸 수 있게 지정하고, 대표 어트리뷰트(부서 필수, 담당자/시스템/소요시간·비용·투입인원 선택)를 설정합니다. 지정 값은 이 맵을 링크한 모든 노드에 라이브로 표시됩니다.
- **버전** — 버전별 워크플로 액션(승인 요청, 게시, 회수 등).
- **승인 대기** / **체크아웃 요청** — 이 맵으로 온 요청을 처리합니다.
- **위험 구역** — **소유권 이전**(나는 편집자로 내려감)과 **맵 삭제**(Owner 전용).

## 6. 공지사항 · 알림·승인 · 피드백

### 공지사항

**공지사항** 탭에 현재 게시 중인 공지가 나옵니다. 전체 / 중요 / 일반 필터, 검색, **읽음 처리**를 지원하며 본문은 마크다운으로 표시됩니다.

### 알림·승인

- **승인 탭** — 내가 처리할 검토 큐: 버전 승인, 체크아웃 이전, 권한·공개 범위 요청. 항목에서 바로 승인/반려(사유 입력)하고 **맵 열기**로 원본으로 이동합니다.
- **알림 탭** — 이벤트 알림이 몇 초 간격으로 갱신됩니다. **모두 읽음** 또는 항목별 읽음 처리.

### 피드백

어느 화면에서든 **피드백**을 클릭해 유형(**버그 / 제안 / 질문 / 기타**)을 고르고 내용을 보내세요 — 현재 화면과 열려 있는 맵이 자동으로 첨부됩니다. 처리 상태와 관리자 답변은 **피드백** 페이지에서 확인합니다.

## 7. 유저그룹

그룹을 쓰면 여러 사람에게 맵 권한을 한 번에 부여할 수 있습니다.

- **그룹 요청**(설정 → 그룹): 이름, 설명, **멤버 2명 이상**, **매니저 1명 이상**(★)을 입력합니다. 관리자가 승인해야 그룹이 활성화됩니다.
- 매니저는 멤버 추가/제거, 매니저 지정, 이름 변경(주 1회), 비활성/재활성, 그룹 삭제를 할 수 있습니다. 삭제된 그룹은 7일간 휴지통에 보관됩니다.
- 내가 속하거나 관리하는 그룹만 보입니다.

## 8. 자주 묻는 질문

- **편집이 안 돼요.** 다른 사람이 체크아웃 중이거나, 승인 진행 중이거나, 내 역할이 Viewer일 수 있습니다. 캔버스 상단 배너를 확인하세요.
- **승인 요청이 안 돼요.** 먼저 결재자를 1명 이상 지정하세요(맵 설정 → 결재자).
- **새 버전을 못 만들어요.** 현재 초안 사이클을 게시까지 끝내세요 — 초안은 한 번에 하나만 존재할 수 있습니다.
- **맵을 실수로 삭제했어요.** 설정 → 삭제 예정에서 7일 이내에 복구하세요.
- **저장이 안 돼요.** 저장 체크리스트를 확인하세요: 시작 노드 1개, 대표 끝 1개, 종료 이름 중복 없음, 일반 노드 다중 출력 없음. (자세한 내용은 **맵 편집** 매뉴얼의 "저장과 검증".)
- **비용을 원·달러 둘 다 못 넣어요.** 회당 비용은 한 통화만 씁니다 — 한쪽을 비운 뒤 다른 통화를 입력하세요.
