# Excel 출력 양식 개선 1안 — 구조 노드 정리 + 분기 주석 (2026-07-17)

## 목표

엑셀 산출물(Export Excel)을 2가지로 나누는 작업의 1단계. 현재 양식을 유지하되 읽기 노이즈인
구조 노드(start/end/무라벨 디시전) 행을 정리하고, 분기 출처를 대상 노드 이름에 주석으로 표기한다.
2안(별도 산출물)은 추가 요구사항 수령 후 별도 설계 — 이번 작업에 플래그·분기를 만들지 않는다.

## 배경 (현재 양식)

- `frontend/src/lib/excel-export.ts` — `buildExcelModel`(순수 모델, vitest 검증) + `writeExcelSheet`(exceljs 기록) 분리.
- 단일 시트, 16컬럼(No~Next), 서브프로세스 재귀 인라인(들여쓰기+outline), 순환/권한/2000행 상한은 노트 행.
- 행 순서는 `orderNodesByFlow`(start부터 BFS), No는 `writeExcelSheet`의 지역 카운터.

## 규칙 (사용자 확정)

| # | 규칙 | 판정 | 처리 |
|---|------|------|------|
| 1 | 무라벨 디시전 행 삭제 | 나가는 엣지가 **전부** 무라벨인 decision (단순 병렬 분기로 간주) | 행 미생성. 혼합 라벨(일부만 라벨)은 행 유지 |
| 2 | 첫 start 외 삭제 | 루트 스코프 BFS 기점 start 1개만 유지 | 서브프로세스 내부 start·추가 start 행 미생성. 제목 무관 |
| 3 | 기본 end 삭제 | title이 trim 후 `"end"` 대소문자 무시 일치 | 행 미생성. 커스텀 제목 end(예: "반품 종료")는 유지 |
| 4 | 분기 주석 | 라벨 있는 디시전의 라벨 엣지 대상 | 대상 Name 뒤 `[<디시전 최종 No>:<엣지라벨>]`. 복수 디시전 대상이면 연접(`[3:yes][7:no]`) |

### Next 컬럼 처리 (사용자 확정)

- **삭제된 end 참조는 텍스트 유지** — `Pack & ship → End`처럼 종착이 읽히게 남긴다.
- **삭제된 무라벨 디시전은 flow-through** — 이를 가리키던 노드의 Next에 디시전의 대상들을 대신 기재.
  연쇄 무라벨 디시전은 재귀 통과. 라벨 디시전 → 무라벨 디시전 경유 시 라벨은 최종 대상까지 전파
  (Next 표기와 규칙 4 주석에 동일 적용).
- **디시전 행 자체의 Next는 기존 그대로** — `대상:라벨;...` 유지 (규칙 4와 정보 중복 허용).

### 부수 결정

- 삭제 대상 노드에 파라미터·설명이 있어도 규칙대로 행 제거 (start/end/무라벨 디시전 = 구조 노드).
- 규칙 4 주석 대상 행이 삭제된 경우(라벨 엣지 → 기본 end 등) 주석은 소멸.
- 노트 행(circular/denied/rowLimit)·2000행 상한·들여쓰기/outline·컬럼 구성·서식 무변경.
- **CSV 내보내기는 미적용** — csv-import와의 왕복(round-trip) 계약이라 구조 노드를 지우면 재임포트가 깨진다.

## 구현 접근 (B안 채택)

후처리(A안)는 `ExcelNodeRow`에 노드 식별자가 없어 제목 매칭에 의존 — 중복 제목·다이아몬드
이중 인라인(같은 맵 2회)에서 오염된다. 엣지 정보가 살아있는 emit 내부에서 처리한다:

1. **삭제 규칙은 emit 시점에 행 미생성** — 규칙 1~3 해당 노드는 rows에 push하지 않는다.
   (서브프로세스 재귀 진입은 유지 — 삭제는 "행"에만 적용.)
2. **Next 계산에 flow-through** — 대상이 삭제되는 무라벨 디시전이면 그 디시전의 대상들로 치환(재귀).
3. **스코프별 nodeId→행 객체 매핑** — 엣지는 스코프(맵 인스턴스) 내부에만 존재하므로, 한 스코프
   emit이 끝나면 라벨 디시전의 각 라벨 엣지에 대해 (디시전 행 객체, 라벨)을 대상 행 객체에 기록.
   행 **객체 참조**라 다이아몬드 이중 인라인·역방향 분기(대상이 디시전보다 앞 행)에도 안전.
4. **번호 부여를 모델로 이동** — `ExcelNodeRow`에 `no: number` 추가. 전체 emit 종료 후 노드 행에
   순번 부여 → 기록된 참조를 풀어 `title`에 `[no:label]` 주석 조립. `writeExcelSheet`는 지역
   카운터를 버리고 `row.no`를 사용.

## 테스트 (vitest, 모델 레벨)

1. SP 내부 start/기본 end 행 미생성, 루트 start 유지
2. 루트 기본 end 삭제 + 커스텀 제목 end 유지 (대소문자/trim 변형 포함)
3. 무라벨 디시전 삭제 + 선행 노드 Next flow-through (연쇄 무라벨 디시전 포함)
4. 혼합 라벨 디시전 행 유지 + 라벨 엣지만 주석
5. 역방향 분기(대상이 디시전보다 앞 행) 주석 번호 정확성
6. 복수 디시전 대상 주석 연접
7. 삭제 행 대상 주석 소멸
8. No 재부여 연속성 (삭제 후 1..n)
9. 다이아몬드 이중 인라인(같은 맵 2회) 주석·번호 무오염
10. 기존 테스트 회귀 — start/end 포함 픽스처 기대값 보정
