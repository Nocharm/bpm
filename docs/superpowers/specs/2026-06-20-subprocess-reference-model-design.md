# 하위프로세스 참조 모델 (Call Activity) — 설계문서

작성일: 2026-06-20 · 상태: 승인됨(브레인스토밍) → 구현계획 대기

## 1. 배경 · 목표

현재 하위프로세스는 **한 버전 안에서 `parent_node_id`로 노드를 중첩**하고, 그 하위 캔버스를 인라인으로 펼쳐 **직접 편집**한다. 최근 버그는 거의 전부 이 *인라인 계층 편집*(childNodes 편집·스코프 분리 저장·좌표 변환·포커스 편집)에서 발생했다.

이를 **"프로세스 참조(Call Activity)" 모델**로 전환한다:

- **편집은 루트 프로세스에서만** 한다. 하위는 **읽기전용**으로 펼쳐 본다 — 하위가 펼쳐져 있어도 루트 편집이 자유롭다.
- 하위프로세스는 **시스템 내 다른 프로세스를 링크**해 가져온다(재사용 가능한 단위).

**기대 효과:** ① 편집 복잡도 제거로 버그 클래스 소멸 ② 코드 대폭 축소 ③ BPMN Call Activity 패턴의 재사용성·개념 명료성.

**비목표(유지):** 이번에 힘들게 만든 **렌더 폴리시**(조상 감싸기·노드 크기 불변·자연스러운 이동·"활성 영역만 이동"·카메라 보정)는 **읽기전용 렌더로 전부 유지·재사용**한다. 다시 작업하지 않는다.

## 2. 결정 요약 (확정)

| 항목 | 결정 |
|------|------|
| 마이그레이션 | **클린 브레이크** — 기존 중첩 데이터 폐기, `parent_node_id` 계층 제거, 데모 재시드 |
| 버전 참조 | **고정 + 알림 기본** + 노드별 **자동추종 토글** |
| 다중 끝 연결 | **대표 끝(primary)** = 주 흐름(버전업 유지) + 나머지 끝 = best-effort(깨질 수 있음) |
| 생성 UX | **프로세스 라이브러리 드래그** |
| 펼침 렌더링 | **A. 인라인 읽기전용 임베드** (렌더 폴리시 유지, 편집만 제거) |

## 3. 데이터 모델

> 기준: `backend/app/models.py` — `ProcessMap`, `MapVersion`, `Node`, `Edge`, `Group`, `Comment`.

### 3.1 Node
- **제거:** `parent_node_id`(계층). 모든 노드는 한 `version_id` 안에서 **평면**.
- **노드 타입 추가:** `node_type` 에 `"subprocess"` 추가. (기존: start / process / decision / end)
- **하위프로세스 노드 필드 추가:**
  - `linked_map_id: str | None` — 참조 대상 프로세스(맵) id
  - `follow_latest: bool = False` — 자동추종 토글
  - `linked_version_id: int | None` — `follow_latest=false`(기본)일 때 **고정 버전** id. `follow_latest=true`면 이 값은 무시하고 렌더 시 그 맵의 **최신 발행(Published) 버전**을 해석한다.
- **끝 노드 필드 추가:** `is_primary_end: bool = False` — 대표 끝(프로세스당 정확히 1개).
- `source_node_id`(복제 계보)는 유지 — 버전 간 끝/노드 동일성 매칭(특히 대표 끝 유지)에 사용.

### 3.2 Edge
- 다중 출구를 위해 **핸들 식별** 추가: `source_handle: str | None`, `target_handle: str | None`.
  - 하위프로세스 노드의 출력 핸들은 **링크된 프로세스의 끝**별로 존재.
  - 대표 끝 핸들 id = 고정 키(예: `"__primary__"`) — 끝 이름이 바뀌어도 `is_primary_end`로 재해석되어 **유지**.
  - 나머지 끝 핸들 id = **끝 이름**(프로세스 내 유니크). 버전업으로 그 이름의 끝이 사라지면 해당 엣지는 **고아(깨짐)** → 경고.
- 기존 `source_side`/`target_side`(시각 변)는 유지.

### 3.3 유효성 규칙(프로세스 단위)
- **시작 노드 정확히 1개**(필수).
- **끝 노드 N개**, **끝 이름 유니크**(중복 불가).
- **대표 끝 정확히 1개**(끝이 있으면). 미지정 시 첫 끝을 기본 대표로.
- 검증은 API 경계(`rules/common/security.md`)에서 수행.

## 4. 연결 규칙

- 하위프로세스 노드: **입력 핸들 1개**(→ 링크 프로세스의 시작) + **끝마다 출력 핸들**.
- **대표 끝 핸들 = 주 흐름** → 부모의 다음 노드. 버전업에도 **확실히 유지**.
- **나머지 끝 핸들 = best-effort:** 부모 작성자가 루트 노드를 그 끝(핸들)에 연결할 수 있으나, 버전업으로 끝이 바뀌면 끊길 수 있음(경고 배지).
- **순환 참조 금지:** 링크 그래프에 사이클(A→B→…→A) 형성 금지. **링크 생성 시점에 검증·차단**(참조 클로저 계산). 자기 자신 참조도 금지.

## 5. 버전 참조 · 업데이트 흐름

- **기본(고정):** 하위프로세스 노드는 `linked_version_id` 고정. 그 맵에 **더 최신 발행본**이 있으면 노드에 **"업데이트?" 배지 + 알림** 표시.
- **자동추종 토글:** `follow_latest=true` 면 렌더 시 항상 그 맵의 **최신 발행(Published) 버전** 해석.
- **업데이트 적용 시:** 새 버전으로 `linked_version_id` 교체. 끝 구성이 바뀌었으면(이름 추가/삭제/변경) **깨진 분기 연결을 표시**해 재연결 유도. 대표 끝은 `is_primary_end`로 매칭되어 유지.

## 6. 렌더링 · 네비게이션 (읽기전용, 폴리시 유지)

- 하위프로세스 노드 펼침 = **링크된 프로세스의 (해석된 버전) 그래프를 로드해 읽기전용 인라인 임베드**.
- **유지·재사용(코드 그대로 활용):** 조상 감싸기(`ancestorContextNodes`), 노드 크기 불변(auto-size), 자연스러운 이동, "활성 영역만 이동"(스코프 네비 시 펼침 유지), 카메라 보정(`focusCamRef`), 깊이별 레인.
- **제거:** 하위 내 편집 경로 — childNodes 편집·`onNodesChange` 분배·자식 드래그/그룹/드롭존·추가/삭제·스코프 분리 저장(`getGraph→변형→PUT`)·`childOffsets`/`scopeOffsets` **편집** 변환.
  - ※ 읽기전용 표시에 필요한 **합성·배치·좌표 변환**은 남는다(편집만 제거).
- **순차 펼침(규칙 3):** 한 단계씩만 열 수 있고, 부모를 닫으면 그 하위는 전부 닫힘 — 항상 순차.
- 읽기전용 임베드의 노드는 `selectable:false`(또는 선택만 가능, 편집 불가), 클릭/더블클릭은 **다음 단계 펼침**용.

## 7. 생성 UX

- 사이드 패널에 **프로세스 라이브러리**(시스템 내 프로세스 목록). 거기서 캔버스로 **드래그** → 하위프로세스 노드 생성 + 드래그한 프로세스로 **자동 링크**(기본 고정 버전 = 그 맵의 최신 발행본, 없으면 최신).
- 순환을 만드는 프로세스는 라이브러리에서 **비활성/차단** 표시.

## 8. 제거 vs 유지 (코드 축소 범위)

| 제거(편집 복잡도) | 유지(읽기전용 렌더 폴리시) |
|---|---|
| childNodes 편집·`onNodesChange` 분배 | 인라인 합성/배치 렌더 |
| 자식 드래그·그룹·드롭존·정렬 | 조상 감싸기·깊이별 레인 |
| 스코프 분리 저장·`childEdits` 오버레이 | "활성 영역만 이동" 네비게이션 |
| `parent_node_id` 계층 CRUD | 카메라 보정·노드 크기 불변 |

→ 순효과: `page.tsx`의 편집 분기 대량 축소, 데이터 모델·저장 경로 단순화.

## 9. 마이그레이션

- 기존 중첩 데모 맵(인라인 펼침 데모·제품 개발 프로세스) **폐기**.
- **새 모델로 데모 재시드:** 평면 프로세스 여러 개 + 그중 일부를 하위프로세스로 링크하는 예시(시작 1·끝 다수·대표 끝·분기 연결·고정/자동추종 사례 포함).
- `parent_node_id` 컬럼·관련 라우터 로직 제거(startup `create_all`이라 마이그레이션 스크립트 불필요, 단 기존 dev.db는 재시드).

## 10. 미해결 · 구현계획에서 정할 것

- 출력 핸들 인코딩 세부(끝 이름 ↔ 핸들 id 정규화, 중복/특수문자 처리).
- "업데이트 알림" 표면(노드 배지 vs 알림 센터 vs 둘 다)과 발행본 비교 방식.
- 프로세스 라이브러리 패널 UI(검색·정렬·순환 차단 표시).
- 읽기전용 임베드에서 대표끝/분기 출구를 부모 캔버스 흐름에 시각적으로 잇는 방식(핸들 위치).
- 검증 에러 UX(시작 2개·끝 이름 중복·대표끝 미지정 시).
