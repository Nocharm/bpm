# 설계 — 담당자/부서 설정 로직 통일 (3개 편집 지점)

> 브랜치 `feat/editor-redesign-r6`. 작성 2026-07-03. R6~R8 재디자인과 별개의 **신규 기능**(데이터 모델·동작 변경).
> 마스터 트래커 `SCREEN-REDESIGN-EDITOR.md`, 커밋 로그 `PROGRESS.md`.

## 1. 한 줄 요약

노드의 **담당자(assignee)·부서(department)** 편집 로직을 3개 지점에서 통일한다: ① 노드 선택 시 우측 인스펙터 속성 탭, ② 노드 편집 모달(`NodeSummaryModal`), ③ 그룹 멤버 속성 일괄(`GroupBulkModal`). 부서는 단일, 담당자는 **같은 부서 내 복수**, 담당자↔부서 연동, 직원 부서 드리프트 시 경고.

## 2. 비목표
- 백엔드/DB 스키마 변경 없음(기존 문자열 필드 재사용).
- 담당자 외 속성(system/duration)은 현행 유지.
- 교차부서 담당자 신규 추가(입력에서 차단하므로 발생 안 함).

## 3. 데이터 모델 (백엔드 무변경)
- `department: string` — 단일 부서명.
- `assignee: string` — **콤마 구분 복수 이름**(예: `"홍길동, 김철수"`). 모두 `department` 소속(입력 시 강제). 벌크 '추가'가 이미 콤마 조인을 쓰므로 호환.
- 조회: `getEligibleAssignees(versionId)` → `users: {id,name,department}[]`, `departments: string[]`. 담당자의 **현재 부서** 및 드리프트 판정의 소스.

## 4. 공용 로직 모듈 `src/lib/assignee.ts` (단일 소스, 3곳 공용)
순수 함수 모음(부작용 없음):
- `parseAssignees(s: string): string[]` — 콤마 분리·트림·빈값 제거.
- `formatAssignees(names: string[]): string` — 콤마(`, `) 조인.
- `deptOf(name, users): string | null` — 인원의 현재 부서(미발견 null).
- `addAssignee(dept, names, name, users): { department; assignees }` — 부서 비면 그 인원 부서로 설정; 부서 있으면 같은 부서 인원만 추가(다르면 무시/거절), 중복 방지.
- `driftedAssignees(dept, names, users): string[]` — 현재 부서 ≠ 노드 부서 **또는 미발견**인 인원 목록(= 경고 대상).

## 5. 동작 규칙 (1·2·3 통일)
- **담당자 추가**: 부서 없으면 그 인원의 부서로 자동 설정. 부서 있으면 피커를 **그 부서로 필터**(교차부서 입력 차단).
- **부서 변경**: 담당자 전체 리셋. 담당자가 있으면 **경고+확인 모달**을 먼저 띄우고, 진행 시 리셋(취소 시 부서 변경 취소).
- **복수 담당자**: 같은 부서 내에서만 추가.

## 6. UI

### 6.1 지점 1·2 — 인스펙터 속성 탭 · 노드 편집 모달
- **부서**: 단일 `SearchSelect`(departments). 변경 시 §5 확인 모달.
- **담당자**: **칩 목록**(각 칩 `×`로 삭제) + **부서필터 `SearchSelect`로 추가**(부서 미설정 시 전체, 설정 시 그 부서만). 드리프트 인원 칩은 **경고색**(`text-error`/틴트).

### 6.2 지점 3 — 그룹 멤버 일괄(`GroupBulkModal`)
- `시스템`/`소요`는 현행(필드 선택 + 값 + 충돌처리) 유지.
- **담당자/부서는 결합된 한 세트**로 편집: 부서 피커 + 담당자 피커(부서필터). 다음 규칙:
  - **부서만 선택(담당자 빔)** → 부서 세팅 = 담당자 리셋이라 충돌 옵션 **대체 / 개별 / 스킵**(추가 비활성 — 부서는 단일).
  - **담당자도 지정** → 4옵션:
    - **대체**: 멤버 담당자=새 값, **부서도 새 담당자의 부서로 교체**.
    - **추가**: 담당자 리스트에 추가. 새 담당자 부서 = 멤버 현재 부서면 그대로 추가; **다르면 부서변경+기존 담당자 삭제가 발생하므로 멤버별 확인 모달**(사실상 개별 체크).
    - **스킵**: 충돌 멤버 그대로.
    - **개별**: 마법사(멤버별 선택).
- 충돌 = 대상 멤버가 이미 담당자/부서를 가진 경우(§ 기존 벌크 충돌 흐름 재사용, 콤마·개별 마법사 포함).

## 7. 경고 (드리프트)
- **판정**: 노드의 담당자 중 `deptOf(name) ≠ node.department` 또는 미발견 → 드리프트.
- **노드 표시**: 드리프트 있는 노드에 **경고 아이콘**(예: `AlertTriangle`) + 편집 UI에서 해당 인원/부서에 경고색.
- **계산 위치**: 페이지에서 `eligible` 1회 로드 → 노드별 드리프트를 계산해 렌더 시 노드 데이터에 `assigneeWarning` 주입(기존 `unresolvedCounts` 주입 패턴과 동일, effect 내 setState 회피).

## 8. 구현 단위 (플랜용)
1. `lib/assignee.ts` 공용 로직 + 단위 테스트(vitest 있으면).
2. 노드 편집 모달(2) 담당자 칩 + 부서 연동 + 부서변경 확인 모달.
3. 인스펙터 속성 탭(1) 동일 적용.
4. 그룹 벌크(3) 결합 세트 + 충돌 옵션 규칙.
5. 드리프트 경고 — eligible 페이지 로드 + 노드 주입 + 노드/편집 UI 경고색·아이콘.

- 단위: `lib/assignee.ts` 순수함수 테스트. 프론트 `lint`·`build` 0. 시나리오: 담당자 추가 시 부서 자동설정 / 부서변경 확인 모달·리셋 / 같은 부서 복수 / 드리프트 경고 / 벌크 4옵션·부서만 3옵션·추가 교차부서 확인.
- 브라우저 픽셀은 사내 Windows/서버(secure-context)에서.
