## 2. 후속 백로그 — 2026-06-25 추가 요청

우선순위 제안: **SR(검색 일괄) · BL(블러) → ST(맵세팅 재구성) · DL(삭제) → PV(퍼블릭 전환) · AP(승인자) · HM(홈)**.
(SR/BL/ST는 다른 항목의 기반이 되므로 먼저.)

### SR. 검색 기능 일괄 개선 (모든 검색 입력에 공통 적용)
대상: 협업자 picker(PrincipalPicker), 승인자 picker, 담당자/부서 `SearchSelect`, 홈 맵 검색, 그 외 모든 검색창.
- **SR-1 키 내비게이션** — 검색 결과에서 `Tab`/`↓`=다음, `↑`/`Shift+Tab`=이전, `Enter`=선택. (현재 선택 항목 시각 표시 필요.)
- **SR-2 유저 검색 키 확장** — 사용자 검색에 **loginId(아이디)도 검색 대상** 추가, 결과에 **아이디·부서 정보 노출**.
- **SR-3 검색엔진 강화** — "순서만 맞으면" 매치(subsequence): 질의 문자들이 **순서대로** 등장하면 노출 + 해당 문자 하이라이트. (`lib/search.ts` 매처 확장 — 현재 부분/초성 매칭 위에 subsequence 추가.)
- **SR-4 위치 고정** — 드롭다운 결과가 늘거나 줄어도 **모달/입력창 위치는 불변**(목록은 overlay/absolute로 띄워 레이아웃 밀림 방지). `SearchSelect`/picker 공통.
- 메모: 공용 검색 UI 컴포넌트로 수렴(현 `SearchSelect`를 일반화하거나 picker와 매처/키핸들러 공유) 검토.

### BL. 모달 바깥영역 블러
- **BL-1** — 모든 모달 백드롭에 블러 효과(`backdrop-blur`) 적용. (ConfirmDialog·PromptDialog는 적용됨 → 컨텍스트 팝업/분기·출력선/액션 모달 등 나머지 점검 및 일관화.)

### ST. 맵 세팅 화면 재구성
- **ST-1 단일 페이지 + 좌측바 내비** — 탭 전환식 대신 **한 페이지에 전 섹션을 쌓고**, 왼쪽바 클릭 시 해당 섹션으로 **스크롤 이동(네비게이션 효과)**. 섹션 사이 **스페이서**로 구분/가시성 확보.
- **ST-2 카드형 그리드** — 좌우로 길어지는 **스택형 버튼/목록**을 **정사각형에 가까운 카드형 그리드**로 교체.
- **ST-3 승인자 카드 그리드** — approvers가 클릭 시 아이디↔이름으로 토글되는 문제 → **아이디·이름을 동시에 보여주는 카드형 그리드뷰**. 로딩 중 **고스트(스켈레톤) 효과**.

### DL. 맵 삭제 플로우 (소프트 삭제 + 복구)
- **DL-1 삭제 모달 강화** — 삭제 확인 모달에 **경고·안내 문구** 추가.
- **DL-2 완료 피드백** — 삭제 후 **"삭제되었습니다" 모달 + 토스터** 표시.
- **DL-3 소프트 삭제 + 복구** — 즉시 삭제 대신 **soft delete(예: `deleted_at`)** 후 **1주일 내 관리자 메뉴에서 복구** 가능. 7일 경과 후 영구 삭제(배치/지연 정리).
  - **백엔드 변경**: ProcessMap에 `deleted_at` 추가, 목록/조회에서 삭제건 제외, 관리자용 복구/영구삭제 엔드포인트, 정리 잡. **결정 필요**: 영구삭제 트리거(주기 배치 vs 접근 시 lazy).

### PV. 퍼블릭 전환 시 뷰어 처리 + 역할 드롭다운
- **PV-1 전환 체크포인트** — 맵 생성 모달에서 **private로 viewer 등록 → 마지막에 public 전환** 시 viewer 설정이 남는 문제. 맵 세팅의 가시성 변경도 동일. **private→public 전환 순간 체크포인트**(잔존 viewer 정리/확인 모달) 필요.
- **PV-2 퍼블릭 역할 드롭다운 제거** — 맵이 public이면 역할 선택 **드롭다운 자체를 제거**(editor만 가능하므로). (F9는 viewer 옵션만 숨겼음 → 드롭다운 통째 제거로 확장.)

### AP. 승인자 지정 대상 제한
- **AP-1** — 승인자(approver) 지정 목록을 **뷰어 권한 이상 보유자 중에서만** 노출. (F5의 `eligible-assignees`와 동일 개념 — viewer+ 자격자 목록을 승인자 picker에도 적용.)

### HM. 홈 화면 (맵 카드 · 상세)
- **HM-1 타이틀 히트박스** — 카드 타이틀 클릭으로 열리는데 **타이틀과 우측 버튼 사이 여백 클릭도 열림** → **타이틀 텍스트로만** 열리게 히트박스 축소.
- **HM-2 소속 하이라이트 버그** — 상세 페이지에서 **팀(부서) 항목이 현재 접속자 소속일 때 하이라이트가 안 됨** → 수정. (개인=loginId, 그룹=내 그룹은 하이라이트되나 부서는 org_path 비교 누락.)
- **HM-3 조직 표기** — 팀(부서) 항목은 **상위 조직을 숨기고 현재(말단) 조직만** 표기(`~Team/~Group` → `~Group`). **노출 순서**: 센터드디파트먼트 / 팀 / 그룹 / 파트 순 내림차순.
