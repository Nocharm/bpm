# BPM 사용 매뉴얼

프로세스맵(순서도)을 그리고, 버전으로 관리하며, 승인 워크플로우로 확정하는 도구입니다. 이 문서는 AI 도우미가 사용법 질문에 답할 때 참고하는 기준 자료이자 사용자 안내서입니다.

## 1. 맵과 캔버스

- **맵 만들기/열기**: 홈 화면에서 **New map**으로 만들고 목록에서 클릭해 엽니다. 생성 시 이름·공개 범위(공개/비공개)·**오우닝 부서(필수)**·**결재자 1명 이상**이 필요합니다(결재자 없이는 승인 요청 불가). 오우닝 부서에 속한 사람은 그 맵의 editor 권한을 항상 가집니다. 맵을 만들면 기본 버전("As-Is")이 함께 생기고 **시작·종료 노드가 자동으로 만들어집니다**. 홈의 "Create from CSV"로 CSV에서 새 맵을 바로 만들 수도 있습니다.
- **노드 추가**: 캔버스 우클릭 → "노드 추가"를 커서 위치에 만듭니다. 노드 유형은 시작(start)/처리(process)/분기(decision)/종료(end)/하위프로세스(subprocess)가 있습니다.
- **노드 속성**: 인스펙터에서 제목·설명·색상·담당자(조직 디렉터리에서 선택)·부서(담당자 따라 자동)·시스템·링크(URL)를 편집합니다.
- **회당 파라미터**: 각 노드는 프로세스 1회 수행 기준의 6개 값을 가집니다 — 회당 소요시간(h)·회당 추가비용(원)·회당 추가비용($)·회당 투입인원·연간 건수·FTE. 소요시간은 "시.분"으로 적고(0.30=30분, 1.30=1시간 30분) `1h30m`으로 표시됩니다. 비용은 원과 달러를 **동시에 넣을 수 없습니다**(한 통화만). 시작·종료 노드는 파라미터가 없고, 하위프로세스 노드는 연간 건수·FTE만 직접 입력하며 나머지는 링크 맵의 지정값을 읽기전용으로 상속합니다.
- **연결**: 노드의 핸들을 끌어 다른 노드에 놓으면 선후(엣지)가 생깁니다.
- **드롭존**: 노드를 다른 노드 근처로 끌면 앞(before)/뒤(after)/자리바꾸기(swap)/그룹으로 넣을 수 있는 타일이 나타납니다.
- **그룹(업무 묶음)**: 노드들을 부서/담당자 단위의 박스로 묶습니다. 그룹 타이틀바에서 이름·색을 바꾸고 전체를 이동할 수 있습니다.
- **노드 복사/복제**: `Ctrl+C`/`Ctrl+V`로 노드(내부 연결 포함)를 복사하고, 다른 탭·다른 맵에도 붙여넣을 수 있습니다. `Ctrl`+드래그로 복제, `Shift`+드래그로 한 축(수평/수직)으로만 이동합니다.
- **하위 프로세스 참조(Call Activity)**: 노드가 다른 맵을 하위 프로세스로 참조하면 그 맵 내용이 **읽기 전용**으로 끼어듭니다(옛 인라인 계층 편집은 폐기됨). 참조는 하위프로세스(subprocess) 노드로만 하며, 기본으로는 서브프로세스로 **지정된 맵**만 참조할 수 있습니다. SP 목록은 우클릭 메뉴나 `S` 단축키로 열고(초성 검색), 같은 맵을 두 번 링크할 수는 없습니다. 새 링크는 기본으로 "최신 따라가기"이며 특정 버전으로 고정할 수도 있습니다.
- **미등록 맵 플레이스홀더**: 맵 이름 드롭다운의 "Show unregistered maps" 토글을 켜면 **지정되지 않은 맵도 링크**할 수 있습니다(경고+잠금 상태로 표시). 링크하면서 대상 맵 오너에게 **등록 요청**을 보낼 수 있고, 오너는 Inbox의 "Subprocess registration" 카드에서 지정 폼을 저장하는 것으로 수락합니다(게시본 필요, 반려 가능). 인스펙터의 Request registration 버튼으로 나중에 요청하거나 본인 요청을 철회할 수 있습니다. 드롭다운 맨 아래 **New map**으로 검색어 이름의 새 맵을 그 자리에서 만들면 현재 맵에 자동으로 링크됩니다.
- **딥 뷰(드릴인)**: 하위프로세스 노드를 더블클릭하면 참조된 맵이 브레드크럼과 함께 읽기 전용 오버레이로 열리고 `Esc`로 나옵니다. (일반 노드 더블클릭은 이름 변경입니다.)
- **정렬**: 자동 정렬로 가로(`Shift+L`, 좌→우) 또는 세로(`Shift+K`, 상→하) 흐름에 맞춰 배치하거나, 다중 선택 후 좌/우/상/하 정렬·등간격 분배합니다.
- **저장**: 변경은 2초 디바운스로 자동 저장되며, 저장 버튼으로 즉시 저장할 수도 있습니다. 저장되려면 저장 체크리스트를 통과해야 합니다 — 시작 노드 1개, 대표 끝 1개, 종료 이름 중복 없음, 일반 노드 출력 1개(다중 출력은 분기 노드만).
- **검색**: /로 노드를 검색합니다(한글 초성 지원). 결과 클릭 시 해당 위치로 이동합니다.
- **가져오기(CSV/AI)**: 상단 바 "가져오기"로 CSV를 올려 노드를 채웁니다. 제목(name)으로 병합하며 빈 칸은 기존 값을 유지합니다. 14개 열(name·description·assignee·department·system·duration·cost_krw·cost_usd·headcount·annual_count·fte·url·url_label·next)을 쓰고, 템플릿을 내려받아 채울 수 있습니다. 홈의 "Create from CSV"로 CSV에서 새 맵을 바로 만들 수도 있습니다.
- **내보내기**: 인스펙터/우클릭 메뉴에서 PNG(이미지), Excel(형식 2종 — 구조 정리형 노드 표 또는 WBS 레벨 표, 통화별 비용 열), CSV(가져오기와 같은 14열), Word(도형 순서도 .docx·노드 링크 하이퍼링크)로 저장합니다.
- **아웃라인**: 좌측 사이드바에 전체 프로세스 트리가 표시됩니다. 클릭하면 해당 노드로 포커싱합니다.

## 2. 버전 관리

- **버전 목록**: 에디터 상단에서 버전을 선택·생성·이름변경·삭제합니다.
- **버전 생성(복제)**: 기존 버전(예: As-Is)을 깊은 복사해 새 라벨로 만듭니다. 이름은 직접 입력하고(자동 "To-Be" 없음) **버전 번호는 게시 시 자동 부여**됩니다. 노드/엣지/그룹이 새 ID로 복제됩니다.
- **맵 이름 변경**: 오너(또는 관리자)는 즉시 변경되고, 편집자가 바꾸면 **변경 요청**이 만들어져 오너가 Inbox에서 승인/반려합니다.
- **비교**: "Compare"로 두 버전을 좌우로 나란히 보고, 추가/삭제/변경을 색으로 확인합니다.

## 3. 승인 워크플로우

버전은 다음 단계로 흐릅니다: **Draft → Pending → Approved → Published** (반려 시 Rejected). 새 버전이 게시되면 직전 게시본은 **Expired(만료·최종 상태)**가 됩니다.

- **승인 대시보드**: 오른쪽 인스펙터 **하단**에 있습니다(노드 선택과 무관하게 항상 표시). 상단에 진행 순서도(stepper), 아래에 승인자 체크리스트(✓승인/○대기)와 진행도, 액션 버튼이 모여 있습니다. 경계를 드래그해 높이를 조절할 수 있습니다.
- **승인자 지정**: 맵 소유자만 대시보드의 "Manage approvers"로 승인자(아이디)를 지정합니다. 승인자가 없으면 제출이 막힙니다.
- **제출(Submit)**: 편집 중(체크아웃 보유)인 작성자가 제출하면 Pending이 되고, 캔버스가 읽기 전용으로 잠기며, 지정 승인자 전원에게 알림이 갑니다.
- **승인/반려**: 지정 승인자가 Approve/Reject(사유 필수)합니다. **전원이 승인해야** Approved가 됩니다(만장일치). 한 명이라도 반려하면 Rejected가 되어 작성자에게 돌아갑니다.
- **게시(Publish)**: 작성자가 Approved 버전을 게시하면 Published가 되고 순차 버전번호(v1, v2…)가 부여됩니다. 같은 맵의 직전 Published는 자동으로 **Expired(만료·최종 상태)**가 됩니다(맵당 게시본 1개). 게시/만료본에서 새 초안을 만드는 **재게시**로 사이클을 다시 시작할 수 있습니다.
- **셀프 게시**: 승인자가 **본인 1명**이면 제출 버튼을 눌렀을 때 "바로 게시할까요?" 팝오버가 떠서 제출→승인→게시를 원클릭으로 끝낼 수 있습니다(No를 고르면 일반 제출).
- **회수(Withdraw)**: Pending/Approved/Rejected 버전을 Draft로 되돌립니다 — Pending/Approved는 **제출자만**, Rejected는 제출자·맵 오너·sysadmin이 회수할 수 있고, 회수자에게 체크아웃(편집권)이 부여됩니다.
- **체크아웃 잠금**: 편집은 한 번에 한 사람만(체크아웃 보유자). 다른 사람이 편집 중이면 읽기 전용으로 보입니다. **체크아웃 요청**(보유자에게 편집권 요청)이나 보유자의 **체크아웃 이전**으로 편집권을 넘기며, **강제 인수는 시스템 관리자만** 가능합니다. 30분 무활동 시 자동 해제됩니다.
- **알림**: 상단 우측 벨에 미읽음 알림이 점으로 표시됩니다(5초 폴링). 항목 클릭 시 알림 탭에서 해당 알림이 열리며 읽음 처리되고, 벨·알림 탭 모두 항목별 삭제(X)가 됩니다. 알림 탭은 선택 삭제·읽은 알림 삭제·날짜 지정 삭제도 지원합니다(모두 되돌릴 수 없음). 알림은 1인당 최근 100건만 보관되며 초과분은 자동 삭제됩니다.

## 4. 코멘트

- 노드 컨텍스트 메뉴의 정보 보기(요약 모달) 또는 인스펙터에서 코멘트를 읽고 작성합니다. 읽기 전용 사용자도 코멘트는 남길 수 있습니다.

## 5. AI 도우미

상단의 "AI assistant" 버튼으로 채팅 패널을 엽니다(서버에서 AI가 활성화된 경우에만 보입니다).

- **순서도 생성**: "구매 발주 프로세스를 그려줘"처럼 지시하면 노드·엣지에 더해 **그룹(부서 묶음)과 담당자/부서/시스템/회당 파라미터**까지 채운 초안을 제안합니다. 담당자/부서는 조직 디렉터리의 실제 인물과 매칭됩니다.
- **부분 편집(증분)**: "검수 단계 뒤에 결재 노드를 추가해줘", "발주 요청의 담당자를 홍길동으로 바꿔"처럼 기존 맵을 고치면 **기존 좌표·색·담당자·그룹을 보존**한 채 해당 부분만 바뀝니다(전체가 새로 그려지지 않습니다). 노드 추가/삭제·연결/연결 끊기·두 노드 사이 삽입("A와 B 사이에 검수 넣어줘")·분기(엣지) 라벨 변경·노드 설명·링크(URL) 설정을 지원합니다.
- **미리보기/적용**: 생성·편집 제안은 캔버스에 **미리보기**로 표시되고, 상단의 **Apply(적용)/Discard(취소)** 로 확정·되돌립니다. 좌표는 자동 배치됩니다.
- **분석·개선점**: "이 맵의 문제점 찾아줘"처럼 물으면 병목·고아 노드·순환·누락 단계·명명 일관성 등을 짚어 **개선점 목록**을 보여줍니다. 항목을 클릭하면 해당 노드로 이동·강조됩니다. 편집 권한이 없어도 됩니다(읽기 전용).
- **단계별 안내(워크스루)**: "이 프로세스 단계별로 설명해줘"처럼 물으면 시작부터 끝까지 노드를 하나씩 따라가며 캔버스가 이동·강조되고 설명이 표시됩니다. 이전/다음으로 넘기거나 자동재생할 수 있습니다(읽기 전용).
- **사용법 질문**: "버전 어떻게 승인해?", "그룹은 어떻게 만들어?"처럼 물으면 AI가 이 매뉴얼을 근거로 답합니다(캔버스는 바뀌지 않습니다). 매뉴얼 밖 내용은 모른다고 답합니다.
- **편집 권한**: 생성·편집(순서도 적용)은 편집 가능한 상태(Draft/Rejected)이고 본인이 체크아웃을 쥔 경우에만 됩니다. 분석·워크스루·사용법 답변은 읽기 전용이라 권한 없이도 됩니다.
- **여러 대화**: 대화는 서버에 저장되어 어느 기기에서든 이어집니다. 대화 바의 목록에서 이전 대화를 열거나 창 헤더의 + 아이콘으로 새 대화를 시작합니다. 대화 제목은 첫 질문에서 자동으로 만들어지고, 목록에서 대화를 삭제할 수 있습니다.
- **다른 맵의 대화**: 대화 목록의 "다른 맵 대화"에서 다른 맵에서 나눈 대화를 읽기 전용으로 열람할 수 있습니다. 이어서 입력하려면 "이 맵 열기" 버튼으로 해당 맵으로 이동합니다.
- **대화 저장과 시간**: 대화는 맵 단위로 서버에 저장되고 메시지마다 시간이 표시됩니다. 보관량은 관리자가 정한 상한(기본: 맵당 대화 20개·대화당 메시지 200개·마지막 활동 후 180일)까지이며, 초과분은 오래된 것부터 자동 정리됩니다. 긴 대화는 최근부터 로딩되고, 스크롤을 맨 위로 올리면 이전 기록이 이어서 로딩됩니다(로딩 중 기능 팁 노출).
- **입력 한도**: 질문은 한 번에 2000자까지 — 입력창 위 오른쪽 링이 남은 분량을 보여줍니다(75% 주의, 90% 경고).
- **글자 크기**: 대화 바의 − T + 버튼으로 채팅 글자 크기를 조절합니다.
- Ctrl+Enter로 전송합니다.
