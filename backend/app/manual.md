# BPM 사용 매뉴얼

프로세스맵(순서도)을 그리고, 버전으로 관리하며, 승인 워크플로우로 확정하는 도구입니다. 이 문서는 AI 도우미가 사용법 질문에 답할 때 참고하는 기준 자료이자 사용자 안내서입니다.

## 1. 맵과 캔버스

- **맵 만들기/열기**: 홈 화면에서 맵 이름을 입력해 생성하고, 목록에서 클릭해 엽니다. 맵을 만들면 기본 버전("As-Is")이 함께 생깁니다.
- **노드 추가**: 캔버스 우클릭 → "노드 추가"를 커서 위치에 만듭니다. 노드 유형은 시작(start)/처리(process)/분기(decision)/종료(end)가 있습니다.
- **연결**: 노드의 핸들을 끌어 다른 노드에 놓으면 선후(엣지)가 생깁니다.
- **드롭존**: 노드를 다른 노드 근처로 끌면 앞(before)/뒤(after)/그룹/하위로 넣을 수 있는 타일이 나타납니다.
- **그룹(업무 묶음)**: 노드들을 부서/담당자 단위의 박스로 묶습니다. 그룹 타이틀바에서 이름·색을 바꾸고 전체를 이동할 수 있습니다.
- **하위 프로세스(드릴다운)**: 노드를 더블클릭하면 그 노드의 하위 캔버스가 창으로 쌓입니다. 상단 브레드크럼으로 계층을 오갑니다.
- **하위 프로세스 참조(Call Activity)**: 노드가 다른 맵을 하위 프로세스로 참조하면 그 맵 내용이 **읽기 전용**으로 끼어듭니다. 참조된 내부는 원본 맵(루트)에서만 편집하고, 참조하는 쪽에서는 바꿀 수 없습니다. 참조는 "최신 따라가기" 또는 특정 버전 고정 중 선택합니다.
- **정렬**: 자동 정렬로 가로(`Shift+L`, 좌→우) 또는 세로(`Shift+K`, 상→하) 흐름에 맞춰 배치하거나, 다중 선택 후 좌/우/상/하 정렬·등간격 분배합니다.
- **저장**: 변경은 2초 디바운스로 자동 저장되며, 저장 버튼으로 즉시 저장할 수도 있습니다.
- **검색**: /로 노드를 검색합니다(한글 초성 지원). 결과 클릭 시 해당 위치로 이동합니다.
- **PNG 내보내기**: 우클릭 메뉴에서 현재 캔버스를 PNG로 저장합니다.
- **아웃라인**: 좌측 사이드바에 전체 프로세스 트리가 표시됩니다. 클릭하면 해당 노드로 포커싱합니다.

## 2. 버전 관리

- **버전 목록**: 에디터 상단에서 버전을 선택·생성·이름변경·삭제합니다.
- **버전 생성(복제)**: 기존 버전(예: As-Is)을 깊은 복사해 새 라벨(예: To-Be)로 만듭니다. 노드/엣지/그룹이 새 ID로 복제됩니다.
- **비교**: "Compare"로 두 버전을 좌우로 나란히 보고, 추가/삭제/변경을 색으로 확인합니다.

## 3. 승인 워크플로우

버전은 다음 단계로 흐릅니다: **Draft → Pending → Approved → Published** (반려 시 Rejected).

- **승인 대시보드**: 오른쪽 인스펙터 **하단**에 있습니다(노드 선택과 무관하게 항상 표시). 상단에 진행 순서도(stepper), 아래에 승인자 체크리스트(✓승인/○대기)와 진행도, 액션 버튼이 모여 있습니다. 경계를 드래그해 높이를 조절할 수 있습니다.
- **승인자 지정**: 맵 소유자만 대시보드의 "Manage approvers"로 승인자(아이디)를 지정합니다. 승인자가 없으면 제출이 막힙니다.
- **제출(Submit)**: 편집 중(체크아웃 보유)인 작성자가 제출하면 Pending이 되고, 캔버스가 읽기 전용으로 잠기며, 지정 승인자 전원에게 알림이 갑니다.
- **승인/반려**: 지정 승인자가 Approve/Reject(사유 필수)합니다. **전원이 승인해야** Approved가 됩니다(만장일치). 한 명이라도 반려하면 Rejected가 되어 작성자에게 돌아갑니다.
- **게시(Publish)**: 작성자가 Approved 버전을 게시하면 Published가 됩니다. 같은 맵의 기존 Published는 자동으로 Approved로 강등됩니다(맵당 공식 버전 1개).
- **회수(Withdraw)**: 작성자는 Pending/Approved 버전을 Draft로 되돌려 다시 편집할 수 있습니다.
- **체크아웃 잠금**: 편집은 한 번에 한 사람만(체크아웃 보유자). 다른 사람이 편집 중이면 읽기 전용으로 보이며, 강제 편집으로 인수할 수 있습니다. 30분 무활동 시 자동 해제됩니다.
- **알림**: 상단 우측 벨에 미읽음 알림이 점으로 표시됩니다(5초 폴링). 클릭해 읽음 처리합니다.

## 4. 코멘트

- 노드를 더블클릭한 요약 모달, 또는 인스펙터에서 코멘트를 읽고 작성합니다. 읽기 전용 사용자도 코멘트는 남길 수 있습니다.

## 5. AI 도우미

상단의 "AI assistant" 버튼으로 채팅 패널을 엽니다(서버에서 AI가 활성화된 경우에만 보입니다).

- **순서도 생성**: "구매 발주 프로세스를 그려줘"처럼 지시하면 노드·엣지에 더해 **그룹(부서 묶음)과 담당자/부서/시스템/소요시간**까지 채운 초안을 제안합니다. 담당자/부서는 조직 디렉터리의 실제 인물과 매칭됩니다.
- **부분 편집(증분)**: "검수 단계 뒤에 결재 노드를 추가해줘", "발주 요청의 담당자를 홍길동으로 바꿔"처럼 기존 맵을 고치면 **기존 좌표·색·담당자·그룹을 보존**한 채 해당 부분만 바뀝니다(전체가 새로 그려지지 않습니다). 노드 추가/삭제·연결/연결 끊기·두 노드 사이 삽입("A와 B 사이에 검수 넣어줘")·분기(엣지) 라벨 변경·노드 설명·링크(URL) 설정을 지원합니다.
- **미리보기/적용**: 생성·편집 제안은 캔버스에 **미리보기**로 표시되고, 상단의 **Apply(적용)/Discard(취소)** 로 확정·되돌립니다. 좌표는 자동 배치됩니다.
- **분석·개선점**: "이 맵의 문제점 찾아줘"처럼 물으면 병목·고아 노드·순환·누락 단계·명명 일관성 등을 짚어 **개선점 목록**을 보여줍니다. 항목을 클릭하면 해당 노드로 이동·강조됩니다. 편집 권한이 없어도 됩니다(읽기 전용).
- **단계별 안내(워크스루)**: "이 프로세스 단계별로 설명해줘"처럼 물으면 시작부터 끝까지 노드를 하나씩 따라가며 캔버스가 이동·강조되고 설명이 표시됩니다. 이전/다음으로 넘기거나 자동재생할 수 있습니다(읽기 전용).
- **사용법 질문**: "버전 어떻게 승인해?", "그룹은 어떻게 만들어?"처럼 물으면 AI가 이 매뉴얼을 근거로 답합니다(캔버스는 바뀌지 않습니다). 매뉴얼 밖 내용은 모른다고 답합니다.
- **편집 권한**: 생성·편집(순서도 적용)은 편집 가능한 상태(Draft/Rejected)이고 본인이 체크아웃을 쥔 경우에만 됩니다. 분석·워크스루·사용법 답변은 읽기 전용이라 권한 없이도 됩니다.
- **여러 대화**: 대화는 최대 4개까지 열 수 있습니다. 대화 바의 목록(카운터 n/4)으로 전환하고, 창 헤더의 + 아이콘으로 새 대화를 시작합니다. 4개인 상태에서 새 대화를 열면 가장 오래전에 연 대화를 닫을지 확인한 뒤 닫고 새로 엽니다. 대화 제목은 첫 질문에서 자동으로 만들어집니다.
- **대화 저장과 시간**: 대화는 브라우저에 저장되고 메시지마다 시간이 표시됩니다. 세션당 최근 40개 메시지까지 보관하며 대화 바 아래 진행바로 사용량을 확인합니다. 긴 대화는 최근부터 로딩되고, 스크롤을 맨 위로 올리면 이전 기록이 이어서 로딩됩니다(로딩 중 기능 팁 노출).
- **입력 한도**: 질문은 한 번에 2000자까지 — 입력창 위 오른쪽 링이 남은 분량을 보여줍니다(75% 주의, 90% 경고).
- **글자 크기**: 대화 바의 − T + 버튼으로 채팅 글자 크기를 조절합니다.
- Ctrl+Enter로 전송합니다.
