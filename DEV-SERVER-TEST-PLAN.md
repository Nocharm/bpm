> **대상 브랜치:** `dev` (origin/dev == dev, 최신 `69ed468`)
> **분기점:** `31a9ea8`(현재 main HEAD) — dev는 여기서 **153커밋 / 17 작업단위** 앞섬. main은 이 시점 이후 미변경이라 dev가 main을 완전 포함.
> **접속:** `http://<서버IP>:3333` (앱 compose nginx). **평문 HTTP = insecure context** → 자동로그인·클립보드·Web Crypto 관련은 **localhost에서 재현 안 되고 서버에서만 검증됨**.

## 0. 배포 델타 (실측 확인 완료)

| 항목 | 내용 | 조치 |
|------|------|------|
| **DB 스키마** | 신규 컬럼 `process_maps.sp_description`(TEXT) 1개 | `db.py` `_ADDED_COLUMNS` 등록됨 → **기동 시 자동 ALTER**. 수동 불필요 |
| **map_rename** | 기존 `approval_requests.kind` 컬럼의 새 값(`'map_rename'`) | 스키마 변경 없음 |
| **신규 env** | `CSV_MANUAL_URL`(선택) — CSV 임포트 안내 버튼 URL | 미설정 시 버튼만 숨김. 기동엔 지장 없음 |
| **compose** | backend `environment:`에 `CSV_MANUAL_URL: ${CSV_MANUAL_URL:-}` 1줄 추가 | `.env`에 값 넣으려면 서버 `.env`에 `CSV_MANUAL_URL=...` 추가(선택) |
| **노드 클립보드** | `localStorage` 기반(평문 HTTP OK) | 없음 |

### 배포 절차
```bash
# 서버(리눅스)에서
git pull origin dev          # 또는 scp/gitlab 경유 — 운영 파이프라인대로
docker compose up -d --build # 접속: http://<서버IP>:3333
```
- **기동 후 backend 로그에서 `ALTER TABLE process_maps ADD COLUMN sp_description` 실행 확인**(기존 DB에 컬럼 없을 때만 1회). 안 뜨면 이미 있는 것.
- ⚠️ **운영 DB 리셋 절대 금지**(`reset_db`=drop_all). 스키마는 자동보강만.
- 배포 전 DB 덤프 백업 권장(자동 ALTER는 안전하지만 관례).

## 3. 브라우저 검증 시나리오

> 권장: **오너 계정 1개 + 일반(편집자/뷰어) 계정 1개** 준비. 승인·이름변경·알림은 역할 전환 검증이 핵심.

### A. 승인 / 버전 워크플로

**A1. 셀프 게시 팝오버 (에디터)** — #13
- [ ] 승인자가 **본인 1인**인 맵에서 편집 → 승인요청 클릭
- [ ] 클릭 지점에 **Yes/No 팝오버** 등장
- [ ] **Yes** → 승인요청→승인→게시가 **원클릭 체인**으로 실행, 새 게시본 생성
- [ ] **No** → 일반 제출(승인 대기)로 처리
- [ ] Cancel → 아무 동작 없음

**A2. 셀프 게시 (설정 Versions 탭)** — #14
- [ ] 설정 > Versions 탭에서 승인요청 → 동일 Yes/No 팝오버
- [ ] **No = 즉시 제출**(에디터와 미세 차이) 확인

**A3. 새 버전 드래프트** — #15
- [ ] 새 버전 만들기 → **이름칸 비어있음**("To-Be" 자동입력 없음)
- [ ] 버전 번호칸 placeholder **"버전 번호는 자동으로 부여됩니다"** 힌트 표시
- [ ] 게시 시 순번 자동 부여 확인

**A4. 맵 이름변경 워크플로** — #16
- [ ] **오너**: 설정에서 이름 변경 → 즉시 반영, 협업자에게 알림, 기존 pending 무효화(supersede)
- [ ] **편집자**: 이름 변경 시도 → 요청 생성(즉시 반영 안 됨), pending 배지
- [ ] **오너 Inbox**: map_rename 카드 노출 → 승인 → 이름 반영 + 토스트 + 요청자에게 알림
- [ ] 요청 취소(withdraw) 동작, 뷰어는 취소 불가(게이트)
- [ ] 소프트삭제된 맵의 rename 요청은 404 정리됨

### B. 에디터 캔버스 UX

**B1. 노드 복사/붙여넣기** — #4,5,6
- [ ] Ctrl+C / Ctrl+V 로 노드+내부 엣지 복사, 라벨에 `(n)` 증분
- [ ] **크로스탭**: 다른 탭/다른 맵에서 붙여넣기(localStorage 클립보드) — **평문 HTTP에서 동작 확인(중요)**
- [ ] 붙여넣기·Ctrl드래그 복제 시 **엣지 핸들 방향 보존**
- [ ] Ctrl+드래그 복제 → 잔상+`+`배지, 다중 노드 **2×N 중복 안 됨**(제스처당 멱등)
- [ ] 추가 직후 노드 **즉시 선택** 상태

**B2. Shift 축 고정** — #4
- [ ] Shift+드래그: 단일 노드가 한 축(수평/수직)으로만 이동
- [ ] 다중 선택 드래그·그룹 이동·선택 사각형에도 축 고정 적용

**B3. 서브프로세스 링크 유일성 / 목록** — #4

**B4. 인라인 펼침 / 읽기전용** — #7,8

### C. 메인 화면 / 대시보드 / 맵 생성

## 4. 서버 전용 함정 체크리스트 (localhost에서 재현 안 됨)

- [ ] **자동 로그인 / 딥링크 복원** — Keycloak `prompt=none`, `returnTo` 복원, 비공개맵 403 모달 (secure context 무관하지만 실 Keycloak 필요)
- [ ] **노드 복사 크로스탭** — `localStorage` 정상(Web Crypto 미사용 확인)
- [ ] **URL/텍스트 복사** — `copyText()` 폴백 동작(평문 HTTP에서 `navigator.clipboard` 부재 시 조용한 실패 없이 처리)
- [ ] **KST 타임스탬프** — 알림·대시보드·로그인 기록 시각이 KST로 정확(postgres)
- [ ] **id 생성** — `genId()`(crypto.randomUUID 금지) 경로에서 노드/엣지 생성 정상
